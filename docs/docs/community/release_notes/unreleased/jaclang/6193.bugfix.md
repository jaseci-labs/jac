- **Fix: Type checker no longer falsely flags attribute access in prefix-chain None guards**: When you write `if obj.attr { use(obj.attr) }` or `[x for x in obj.attr] if obj.attr else []`, the checker used to report E1099 saying `obj` might be `None` even though a truthy `obj.attr` proves `obj` is not `None`. The fix teaches the checker to treat a truthiness guard on any dotted chain as implicitly narrowing all of its non-None prefixes.

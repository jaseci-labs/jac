- **Fix: Lambda param annotations now parse correctly for all annotation forms**: Subscripted types (`list[int]`, `dict[str, int]`), union types (`int | str`), return-hint lambdas (`lambda x: int -> None { body }`), and brace-body lambdas (`lambda x: int { body }`) all parse correctly. A bracket-depth scan replaces the previous two-token speculative lookahead, also fixing regressions where arithmetic, unary, subscript, or dict-literal bodies were misread as type annotations, where the scan incorrectly crossed sibling lambda boundaries in list/tuple contexts, and where the dict key separator `:` inside dict-value lambdas (`{"k": lambda x: x * 2}`) was misidentified as an annotation separator.

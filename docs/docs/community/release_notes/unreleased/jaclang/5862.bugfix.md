- **Fix: `jac jac2js` no longer collapses kwargs into a positional object literal**: calls compiled by `jac jac2js` packed every kwarg into a single object literal that JS receives as the first positional argument, so `add(b=10, a=5)` ran as `add({b:10,a:5}, undefined)` and produced `"[object Object]undefined"`, while `add(1, b=2)` silently dropped the leading positional. The same shape leaked into the sv-import call site whenever the type checker wasn't run (the production CLI path), causing 422s, and the wrapper itself emitted empty params because the source module wasn't pre-loaded into `prog.mod.hub`. `EsastGenPass` now resolves call-site arguments (positional, kwarg, and `**` spread) onto the callee's declared parameter list whenever the signature is statically reachable and emits an ordered positional JS call, with slot resolution shared by the sv-import branch. Sv-import wrapper generation and call-site detection lazy-load the source module via the same `symtab_ir_only` path the type evaluator uses, so the production build path now produces the same JS as the type-checked test path.

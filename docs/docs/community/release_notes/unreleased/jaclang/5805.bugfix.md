- **Fix: jac format whitespace bugs in lambda/if inline blocks (#5779)**: Three follow-up regressions to the #5523 lambda-mashing fix are resolved. (1) An `if` block inside a multi-statement lambda body now expands to multi-line form instead of staying inline with a stray double space (`if cond { stmt;  }`). The flat-capable shortcut is now strictly scoped to lambdas whose entire body is the single if-statement. (2) Inline single-statement code-block lambdas now keep one space between the trailing `;` and the closing `}`, so output is `lambda { stmt; }` instead of `lambda { stmt;}`. (3) `jac format` is now idempotent on the corrected output: running it on its own output produces no diff.

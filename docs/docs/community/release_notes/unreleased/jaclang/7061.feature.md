- **Feature: bundled native `fractions` (#6978 Phase 2)**: `import from fractions { Fraction }` resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` module (`fractions.na.jac`) -- a pure-Jac (Mechanism B) `Fraction` over native `int`, normalized on construction via Euclid's GCD with the sign carried by the numerator and the denominator kept positive, matching CPython's `fractions.Fraction` value model. Construction and reduction (`Fraction(3, 6)` -> `1/2`, sign normalization, whole numbers, zero), the `numerator` / `denominator` properties, and `str()` match CPython exactly. Because the native backend has no operator-overload dispatch yet, arithmetic and ordering are exposed as methods (`add` / `sub` / `mul` / `div` / `eq` / `lt`) rather than `+` / `<`; the resulting values are congruent. Float/Decimal/string construction is out of scope. A new equivalence fixture (`prim_fractions`, registered with `require=["na"]`) pins the native module congruent with CPython's `fractions` across construction, reduction, the four arithmetic operations, and ordering.

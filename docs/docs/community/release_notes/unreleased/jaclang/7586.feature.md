- **Feature: opt-in native-default codespace - `[build] default_codespace = "native"` infers na for markerless modules**: a new jac.toml knob (values `server`/`sv` and `native`/`na`; shipped default stays `server`) lets a project make the native codespace the preferred default for plain `.jac` modules. Under a native default, a markerless module whose parse tree carries no server-requiring signals (OSP constructs, node/edge/walker archetypes, async or event abilities, yield, lambda, inline python, JSX, explicit `sv`/`cl` blocks or markers, non-native imports, sibling variant files) and whose plain-`.jac` import closure is likewise clean is coerced whole-module native, with `sv {}` blocks remaining the per-block escape hatch; anything else compiles exactly as before. Closure classification is a deterministic parse-level walk (annexes included, results mtime-cached) with cycle detection, since the native backend cannot link circular imports. Supporting changes: modules now carry an authoritative `decided_codespace` attribute consulted ahead of file-suffix sniffing (jir FORMAT_VERSION 15 -> 16), the native stdlib import allowlist moved to `jac0core.codeinfo` as the single source of truth, `resolve_native_module` gained a classifier-gated plain-`.jac` fallback so inferred-native modules can import each other, and `jac run` executes classified modules through the native engine with transparent server fallback. A `CompileOptions.default_codespace` override serves embedders and tests.

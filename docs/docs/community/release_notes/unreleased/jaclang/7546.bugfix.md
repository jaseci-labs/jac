- **Fix: Emit @jac/runtime imports for client JS**: Codegen now injects `__jacJsx` and `__jacSpawn` imports through the ESAST import mechanism so `mod.gen.js` is emission-ready without consumer-side finalizers.
- **Fix: React/Preact JSX producer imports**: `ReactBackend.view_runtime_symbols()` now returns the JSX factory symbol so ordinary JSX elements get compiler-owned imports.
- **Fix: Backend-neutral compiler fragments**: Compiler-created fragments lower through `ViewElement`/`FragmentTag` IR so Solid keeps native `<>...</>` fragments instead of importing `__jacJsx`.
- **Fix: npm publishing duplicate runtime imports**: The npm collector remaps `@jac/runtime` to the configured runtime package instead of prepending a second import.
- **Fix: Runtime npm tarball self-import**: Building `@jac/runtime` packages no longer prepends consumer runtime imports onto the runtime shim itself.

- **Feature: bundled native `textwrap`, `csv`, `pprint`, and `difflib` (pure-Jac, #6978 Phase 3)**: four more standard-library modules now resolve on the native (`nacompile`/JIT) pathway to bundled `na_stdlib` Mechanism-B modules -- ordinary Jac compiled like user code, no per-module backend work, portable to every native target. The same `import from <mod> { ... }` binds CPython on the `sv` (Python) backend and the bundled `.na.jac` on the `na` (native LLVM) backend, and each is pinned `sv`<->`na` congruent by a new tri-backend equivalence fixture (`prim_textwrap`, `prim_csv`, `prim_pprint`, `prim_difflib`). `textwrap` ports CPython's greedy `TextWrapper` wrapper (`wrap`/`fill`) plus `dedent`/`indent` (`break_on_hyphens=False`, all other defaults matched; `shorten`/`TextWrapper` out of scope). `csv` provides `reader` for the default **excel** dialect with exact field-level parsing (quoted fields, doubled quotes, mid-field literal quotes, unterminated quotes, blank line -> empty record), returning an eager `list[list[str]]` congruent with `list(csv.reader(...))`. `pprint` provides `pformat` rendering a single-line, sorted-key Python `repr` (congruent for objects that fit CPython's default `width=80`; multi-line wrapping and floats out of scope). `difflib` ports `SequenceMatcher` (`ratio`, `get_matching_blocks`, `set_seq1`, `set_seq2`) and `get_close_matches`, with `ratio` returning the same IEEE-double value as CPython; `isjunk` is accepted but ignored and there is no autojunk pruning, so the two are congruent for the sequences shorter than 200 elements where CPython's autojunk never engages. Documented divergences are recorded in `na_stdlib/README.md`.

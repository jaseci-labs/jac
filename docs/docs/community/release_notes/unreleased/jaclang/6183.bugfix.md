- **Fix: a single bad field value no longer takes down your entire node graph**: Previously, if one field on a node had `null` stored in the database — caused by a schema migration, a past write bug, or a removed nested class — the runtime would quarantine that node and cascade-quarantine every connected edge, wiping out thousands of unrelated nodes in one go. The fix makes the runtime recover gracefully: it resets the bad field to the value declared in your code (e.g. `has things: list[X] = []` comes back as `[]`) and loads the node normally, with a warning in the logs so you know it happened. If a field genuinely has no declared default and cannot be recovered, the node is still quarantined — but only that one node, not its neighbours. Additionally, both `SqliteMemory` and `MongoBackend` now write a precise quarantine reason (`"class X unresolvable"` vs `"archetype field deserialization failed: X"`) so when something does go to quarantine, you can tell immediately whether to look at your import paths or your field types.

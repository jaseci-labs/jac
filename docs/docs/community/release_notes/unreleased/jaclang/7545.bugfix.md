- **Fix: `jac lint` report path no longer runs the full AST-rewriting pass**: `get_lint_sched()` (behind `jac lint` and `JacProgram.jac_file_linter`) returned `[JacAutoLintPass]` with `report_only=False`, so the report path executed the full AST-mutating auto-lint pass and then discarded the rewritten module; only the warnings/errors are ever read. It now returns `[JacLintCheckPass]` (`report_only=True`), matching what `jac check` already uses. Identical warnings across all 1414 tracked `.jac` files, ~6x faster for the lint pass, and ~19% faster for `jac lint` end-to-end.

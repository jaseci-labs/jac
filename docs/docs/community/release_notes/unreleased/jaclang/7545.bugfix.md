- **Fix: `jac lint` report path no longer runs the full AST-rewriting pass**: `get_lint_sched()` (behind `jac lint` and `JacProgram.jac_file_linter`) returned `[JacAutoLintPass]` with `report_only=False`, so the report path executed the full AST-mutating auto-lint pass and then discarded the rewritten module; only the warnings/errors are ever read. It now returns `[JacLintCheckPass]` (`report_only=True`), matching what `jac check` already uses: ~6x faster for the lint pass and ~19% faster for `jac lint` end-to-end. The report-only pass was also under-reporting: several autofix-class diagnostics (empty parens, combine-`has`/`glob`, `staticmethod`-to-`static`, `__init__`-to-`init`, keyword-escape, unnecessary `from __future__ import annotations`, import semicolons, ternary/getattr/hasattr rewrites, `useState`-to-`has`, map-to-comprehension, property conversions) were emitted only on the mutation path and silently dropped when reporting. They are now surfaced in both `jac lint` and `jac check`. The only remaining divergence is *follow-on* autofixes the mutation pass can only see after rewriting the tree (e.g. combining `glob`s it just extracted from a `with entry`, or simplifying a ternary once its `hasattr` condition is rewritten); those surface on the next lint after the primary fix is applied.

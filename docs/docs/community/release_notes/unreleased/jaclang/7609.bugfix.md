- **Fix: an unreachable `MONGODB_URI` no longer hangs `jac run` and `jac start`**: Mongo clients are now built with a bounded `serverSelectionTimeoutMS` (new `mongodb_timeout_ms` database config key, default 5000, overridable via `MONGODB_TIMEOUT_MS` or `jac.toml`), and the persistence and identity layers each probe once before committing to Mongo instead of discovering the failure one blocking round-trip at a time. Previously a stale URI left `jac run` blocked for two minutes and `jac start` never binding a port; both now fall back to local storage in seconds with a warning naming the URI (credentials redacted).

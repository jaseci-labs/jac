- **Refactor: `@hookable` plugin hooks and domain-organized runtime interface**: The runtime interface now marks its 20 actual plugin extension points with a `@hookable` decorator at their declaration; only those go through `plugin_manager` dispatch, while the other ~83 methods are plain inherited methods called directly. `generate_plugin_helpers` (which reflected over every method and regenerated the whole `JacRuntimeInterface` proxy each load) is replaced by `build_plugin_hook_layer`, which wires up only the hookable methods and installs dispatching proxies on a now-stable, non-reassigned interface class. The flat mixin set (including the 31-method `JacBasics` grab-bag and `JacUtils`) is re-decomposed into cohesive domain classes (`JacContext`, `JacPersistence`, `JacGraph`, `JacArchetype`, `JacModule`, `JacIntrospection`, `JacLLM`, `JacServe`, `JacTest`, `JacBuiltin`, `JacConfig`, plus the kept `JacAccessValidation`/`JacWalker`/`JacCmd`/`JacClassReferences`). Behavior is unchanged.

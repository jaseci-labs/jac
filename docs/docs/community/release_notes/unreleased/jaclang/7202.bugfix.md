- **Mongo "ready" now means authenticated-ready, and pods no longer crash-restart into its boot window**: the mongo pod gains a readinessProbe that runs an authenticated ping, and since its headless service only publishes ready pods to DNS, the app pods' wait-for-mongodb init container now waits for a mongod that accepts authenticated commands instead of just an open TCP port. As defense-in-depth for mid-life mongo restarts, the first handshake in a scale runtime retries transient connection failures with a bounded budget (15x2s); auth and config errors raise immediately, so bad credentials fail fast instead of hanging through the budget. Local CLI use is unaffected (gated to `JAC_SCALE_RUNTIME`).

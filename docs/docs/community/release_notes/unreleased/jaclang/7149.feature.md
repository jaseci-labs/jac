- **Opt-in ownership & borrow checker, plus a native cycle collector (#7149)**: A new, opt-in ownership layer on top of Jac's existing managed (RC/GC) memory. Unannotated code is unchanged; you enter the owned world only by writing annotations. `buf: own Buffer` marks a move-checked binding, `view: &buf` takes a shared (immutable) borrow, and `edit: &mut buf` takes an exclusive mutable borrow. A new `OwnershipCheckPass` walks the control-flow graph after `CFGBuildPass` and enforces the rules with five diagnostics, on every backend: `E1301` use of a value after it was moved (assigning it elsewhere or sealing it into managed storage moves it out); `E1302` a mutable borrow taken while another borrow is live (a value may have many shared borrows or one mutable borrow, never both); `E1303` mutating an owner while a shared borrow of it is live; `E1304` an owner destroyed while still borrowed; and `E1306` a borrow that escapes its scope (borrows are second-class, so a reference may not be stored in a field, returned other than as a single passthrough parameter, or otherwise outlive the scope that created it). Ownership is affine, not linear: an `own` binding that is never consumed is simply dropped and reclaimed by the managed RC/GC floor -- that is not an error. `E1305` is reserved (defined but unemitted) for a future `linear` resource marker that would require an explicit move. `&mut` exclusivity is proven from flow analysis, not from trusting the annotation. Only `obj` and value/primitive types are ownable; `node`/`edge`/`walker` graph types stay permanently managed. The checking runs on all backends; the native backend adds two payoffs: an opt-in synchronous Bacon-Rajan cycle collector so reference cycles in managed ARC memory can be reclaimed rather than leaked, and RC-elision for proven-unique owned values, where a move (`b = a`) of a consumed, never-borrowed binding is lowered without the defensive retain (the source slot is nulled so its later cleanup release is a no-op and the object is freed exactly once). The cycle collector's automatic sweep is gated behind the `JAC_GC_CYCLES` environment variable and is off by default, so native programs run on plain reference counting; set `JAC_GC_CYCLES=1` to enable automatic cycle reclamation.

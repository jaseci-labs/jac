- **Fix: type checker narrowing for `in`/`not in` and string-literal comparisons**: `x in (a, b)` / `x not in (a, b)` now narrows away `None` when every element is a literal, and `x == "literal"` / `x != "literal"` now narrows correctly for string literals (it previously only worked for `int`/`float`/`bool`). Both were silent false-positive sources for `jac check`; no user-facing behavior changes beyond fewer spurious type errors.

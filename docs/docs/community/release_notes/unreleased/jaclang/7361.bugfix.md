- **Type Checker: Enum metaclass attributes resolve correctly**: accessing `__members__`, `_member_names_`, `_member_map_` or `_value2member_map_` on an enum class no longer raises a false `error[E1030]: has no attribute`. These live on the enum metaclass (`EnumType`), not in the enum's own MRO, so the checker now synthesizes their types (`__members__`/`_member_map_` as `dict[str, Enum]`, `_member_names_` as `list[str]`, `_value2member_map_` as `dict[value, Enum]`). Member-level access such as `Color.RED.__members__` stays correctly rejected.

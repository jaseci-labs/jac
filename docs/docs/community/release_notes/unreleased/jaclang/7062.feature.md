- **Feature: bundled native `bz2` over a bundled `libbz2` floor (#6978 Phase 2)**: `import from bz2 { compress, decompress }` resolves on the native (`nacompile`/JIT) pathway to a bundled `na_stdlib` module -- a thin pure-Jac surface (`bz2.na.jac`) over the system `libbz2`, reached through an FFI floor (`_bz2_native.na.jac`) and statically linked from jac's shipped `libbz2.a` archive, exactly as CPython's `bz2` wraps the same library. The BWT engine is never reimplemented. Coverage is the one-shot buffer API: `compress(data, compresslevel=9)` produces a single bzip2 stream byte-identical to CPython's (same default `workFactor`), and `decompress(data)` round-trips a single stream, growing its output buffer on `BZ_OUTBUFF_FULL`. The floor wraps `BZ2_bzBuffToBuffCompress` / `BZ2_bzBuffToBuffDecompress`; note `libbz2` takes the in/out `destLen` as `unsigned int*` (a 4-byte cell), unlike zlib's 8-byte `uLongf*`, so the surface allocates a 4-byte length buffer. This is the first link of the bundled `libbz2.a` floor archive. SCOPE: the one-shot buffer API only -- `decompress` handles a single stream (CPython additionally concatenates multiple streams), and incremental `BZ2Compressor` / `BZ2Decompressor` and the file API are out of scope. A new equivalence fixture (`prim_bz2`, registered with `require=["na"]`) pins the native module congruent with CPython's `bz2` on compressed-stream byte-identity (poly digest + exact length, at default and explicit levels), round-trip, and empty input.

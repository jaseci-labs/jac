- **Feature: k8s deploy ships a prebuilt `jac` binary, driven by `[dev]`**: the deploy no longer builds jaclang (host or in-pod). It always downloads a prebuilt, self-contained binary from jaseci-labs releases and ships it to pods over the bundle PVC. The project's `[dev]` jac.toml stanza picks the runtime: no `[dev]` runs the latest **stable** release as-is; a bare `[dev]` runs the rolling **dev** prerelease (built from `main` HEAD) as-is - no source overlay, because it already is `main`. Overlaying a jaclang source tree only happens when you request a custom source: `[dev] repo`/`branch`, or `[dev] jaclang_source` pointing at a local `jac/` checkout (else fetched `main`). The overlay is applied via the editable dev loop and the pod compiles it once on first boot. scale and the MCP server are built into the binary, so pods never clone, install plugins, or build.

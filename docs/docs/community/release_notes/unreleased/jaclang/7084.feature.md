- **Feature: k8s deploy ships a prebuilt `jac` binary, driven by `[dev]`**: the deploy no longer builds jaclang (host or in-pod). It always downloads a prebuilt, self-contained binary from jaseci-labs releases and ships it to pods over the bundle PVC. The project's `[dev]` jac.toml stanza picks the runtime: no `[dev]` runs the latest **stable** release as-is; `[dev]` uses the rolling **dev** prerelease and overlays a jaclang source tree on top via the editable dev loop (`jaclang_source`) - from `repo`/`branch` if given, else the local `jac/` checkout if present, else fetched `main`. When the deploy host's arch matches the node's, the overlay is precompiled to `.jir` with the downloaded binary for instant pod boot; otherwise the pod compiles it once on first boot. scale and the MCP server are built into the binary, so pods never clone, install plugins, or build.

- **Fix: vite dev polling watcher no longer reloads on runtime state changes**: The generated `vite.dev.config.js` watched the build root with `usePolling` and no `ignored` list, so the desktop runtime's SQLite anchor store under `<root>/.jac/data/` (and its `-wal`/`-shm` sidecars) retriggered HMR on every DB write — a full-reload loop for any `--client desktop --dev` app that touched its DB. Added a `server.watch.ignored` list covering `.jac/data/**` and `*.sqlite`/`*.db` files; Vite concatenates it with its built-in defaults, so the change is purely additive.

- **Feature: editable jaclang dev loop (`[dev] jaclang_source`)**: the self-contained `jac` binary runs its own bundled copy of jaclang, so edits to the in-repo `jac/jaclang` source previously required a full `zig build` to take effect. Declaring `[dev] jaclang_source = "<dir containing jaclang/>"` in the nearest `jac.toml` now reroutes `import jaclang` to that source tree -- `sitecustomize` prepends it to the front of `sys.path` at startup (before the launcher boots jaclang), so compiler/pass/CLI/runtime edits run live with no rebuild. The override sets `JAC_NO_PRECOMPILE=1` to skip the shipped precompiled JIR bundle; the per-module cache is content-keyed, so edits self-invalidate. A cheap substring guard keeps non-dev startup free of the `tomllib` parse. Requires a one-time `zig build`, since `sitecustomize.py` ships inside the binary payload.

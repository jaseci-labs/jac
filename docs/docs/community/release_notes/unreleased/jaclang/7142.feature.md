- **The bundled `jac` binary is fully source-free, booting from a first-class JIR image**: a bundled `zig build` payload no longer ships importable `.jac` (or jaclang `.py`) source — it boots and imports jaclang entirely from a sealed JIR *image* trusted by construction (#7135, #6852 Phase 4). A build-time `_precompiled/MANIFEST.json` maps module fullnames to JIR, and jaclang's own `JacMetaImporter` resolves its modules by name from the manifest **first**, with no filesystem `.jac` probing and no per-load source re-hash; trust moves to build time (the manifest) plus the launcher's existing payload sha256 trailer, and the runtime fail-closes on a manifest/tag/format mismatch instead of degrading to live compilation. Both compile tiers share one container and one manifest: full-compiler modules carry a normal JIR, and the jac0-compiled `jac0core` bootstrap layer (whose JIR reader is itself a `jac0core` module) is carried as `"bootstrap"`-flagged JIR loaded by a pure-Python section reader, so the bootstrap cycle stays broken while the on-disk format is unified. The stdlib and jaclang `.py` ship as sourceless unchecked-hash `.pyc` (PEP 552); pip-installed dev tooling keeps its source. Sealing is the only bundled shape (`mkpayload` always passes `--seal`; `-Ddebug-src` optionally embeds source text in the JIR so tracebacks still show source lines source-free), and it is strict — any precompile failure aborts the build. The same image format powers two user-facing products: `jac bundle --target sealed` emits a portable source-free image, and `jac bundle --target binary` wraps that image onto a copy of the running sealed binary to yield a self-contained executable that runs with zero runtime compilation.

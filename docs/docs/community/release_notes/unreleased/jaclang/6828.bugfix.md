- **Fix: macOS/Windows shared-library exports resolve through `ctypes`/`dlopen`**: The native linker no longer emits an `LC_LOAD_DYLINKER` load command in Mach-O dylibs (it is only valid for executables and could cause `dlopen` to reject the file) and defaults `CS_EXECSEG_MAIN_BINARY`'s `execSegFlags` to 0 for dylibs (the `0x1` main-binary flag is only valid on an executable's `__TEXT` segment and trips a macOS load error when set on a shared library). PE/DLL generation also picks up the matching export and entry-point fixes. Added Mach-O `dlopen`/`dlsym` round-trip and export-trie naming regression tests (macOS-gated) alongside the existing structural load-command guards.

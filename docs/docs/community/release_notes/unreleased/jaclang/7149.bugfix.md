- **Fix: diagnostic help text is now formatted, and borrow expressions type-check**: `DiagnosticInfo.help_text` was rendered verbatim, so templated help strings leaked literal placeholders (e.g. `E1301`'s help showed `'{name}'` instead of the binding name). Help text is now formatted with the same kwargs as the message, guarded so help containing literal braces (such as `E0025`'s `{ ... }` syntax example) falls back to raw. Separately, a borrow expression `&x` / `&mut x` evaluated to `<Unknown>` because the type evaluator's `UnaryExpr` case handled `NOT`/`MINUS`/`BW_NOT` but not unary `&`; it now resolves to the operand's type, eliminating the spurious `E1001`/`E1053` "Cannot assign `<Unknown>`" errors that masked the borrow checker's real `E13xx` diagnostics on every borrow fixture.
- **Fix: a reserved-word local no longer silently disables the whole type checker**: a bare local named `root` in the ownership pass's markers annex shadowed the graph-root keyword and broke codegen for the module, so its meta-import raised "No bytecode found". Because the type-check scheduler swallows that in an `except ImportError` fallback, the failure silently disabled the entire type-check schedule -- every ownership diagnostic (`E1301`-`E1309`) and static analysis went quiet rather than erroring. The local was renamed, restoring the checker.

- **Bugfix: `jac tool jac2js` and the MCP compiler bridge now inject `@jac/runtime` globals**: both standalone emission paths printed/returned raw `ir.gen.js` without calling `prepend_active_runtime_globals()`, so JSX client modules referenced `__jacJsx` (and other `@jac/runtime` globals) with no matching import and threw `ReferenceError: __jacJsx is not defined` when loaded as ESM. Both paths now wrap their output the same way every other emission consumer already does.

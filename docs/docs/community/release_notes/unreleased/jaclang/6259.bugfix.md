- **Fix: `Enum.__members__` and other metaclass attributes no longer raise false E1030**: Accessing `Color.__members__`, iterating with `name in Color.__members__`, or subscripting `Color.__members__["RED"]` previously raised `E1030: Attribute '__members__' not found in type 'Color'` even though the code is valid Python/Jac. The `__members__` attribute (and `__iter__`, `__len__`, `__getitem__`) live on `EnumMeta` (the metaclass), not in the enum class MRO — the type checker only walked the class MRO and never checked the metaclass. The fix teaches the type system about metaclasses end-to-end: the `metaclass=` keyword from Python stub files is now captured, stored in the AST, resolved to a real `ClassType`, and inherited by subclasses (so `Color(Enum)` automatically gets `EnumMeta` as its metaclass). Attribute lookup now falls back to the metaclass MRO when the class MRO misses, and `@property` descriptors found via the metaclass are correctly unwrapped to their return type.

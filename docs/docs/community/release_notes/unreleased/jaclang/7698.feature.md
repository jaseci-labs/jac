- **jac-scale: pin the pod runtime to a published release**: `[scale-runtime] version = "X.Y.Z"` in `jac.toml` (or `JAC_SCALE_VERSION=X.Y.Z` for a one-off deploy) holds `jac start --scale` pods on that exact release instead of the rolling latest - the rollback lever when a regressed stable publishes. One pin moves the runtime binary and admin console (`releases/tags/v<version>`), the pod base image (`jaseci/jaclang:<version>`), and the download cache slot together, and pods carry a `JAC_SCALE_RUNTIME_VERSION` env for verification. Supported pins are the current minor and one back; `JAC_SCALE_BINARY_PATH` still outranks everything as the air-gap escape hatch.

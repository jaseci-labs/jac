- **CI: the LLVMPY_* shim is cached across release builds and published for contributors**: the native backend's LLVM shim (the single most expensive build artifact, a static link of ~0.5 GB of LLVM) is no longer relinked from source on every release and dev build. `release-jaclang.yml` and `release-dev.yml` now restore a content-keyed cache of the built shim and bundle it via `-Dshim-bin`, skipping the LLVM fetch and link on the ~97% of runs where `jac/native/` is unchanged. The same target-pinned shim is published per-platform to a `jacllvm-v<LLVM_VER>` prerelease tag, so `scripts/fresh_env.sh` fetches a sha256-verified prebuilt shim instead of downloading and linking LLVM locally. Shipped binaries are unaffected (the shim is byte-identical; releases do not depend on the tag).

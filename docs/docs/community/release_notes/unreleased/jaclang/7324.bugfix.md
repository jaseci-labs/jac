- **Fix: `jac start -p PORT` is honored for apps with `sv` services**: The scale gateway always bound `gateway_port` (default 8000) and silently ignored `-p`, so a busy 8000 failed with `address already in use` instead of using the requested port. The CLI port is now forwarded to the gateway. Port precedence is `-p` > `[scale.microservices] gateway_port` > 8000 for the gateway, and `-p` > `[serve] port` > 8000 for the single-process server; a bare `jac start` no longer overrides a configured `gateway_port`.
- **Fix: `jac start app.jab` honors the app's configured `[serve] port`**: `jac start` on a sealed `.jab` always bound 8000, ignoring the app's `[serve] port` and diverging from `jac run app.jab`. Both now resolve the jab's configured port, and an explicit `-p` still wins.

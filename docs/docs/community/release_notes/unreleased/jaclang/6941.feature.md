- **Feature: Bundled native standard library (`na_stdlib`) + `os.path`**: the native (`na`) pathway can now ship pure-Jac standard-library modules under `jaclang/runtimelib/na_stdlib/`, so `import from <mod> { fn }` resolves a bundled `.na.jac` module and links it through the existing cross-module machinery with zero per-module backend work: the same source binds CPython's real module on the sv pathway and the bundled one on na. Resolution is **nearest-wins**: the single shared `resolve_native_module` (used by `BoundaryAnalysisPass`, `NaIRGenPass`, and `NativeCompilePass`) searches the importing project's own tree first and falls back to the bundled root only on a miss, so a user module of the same name always shadows a bundled one. `NativeCompilePass`'s link-set discovery gained the matching bundled-root fallback so a bundled dependency's object code (`__jac_glob_init_<mod>`) is compiled and linked into both AOT (`jac nacompile`) standalone binaries and the JIT engine. The first bundled module is **`os.path`** (POSIX): `normpath`/`dirname`/`basename`/`split`/`splitext`/`isabs`, byte-for-byte congruent with CPython `posixpath` (including the collapsed-slash / `..`-resolution / leading-dot edge cases), pinned by the new `prim_ospath.jac` tri-backend equivalence test.

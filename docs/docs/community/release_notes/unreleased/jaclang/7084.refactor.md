- **Perf: bound precompile memory by releasing per-unit AST closures**: the bytecode precompile driver (used to build the bundled `.jir` cache and to warm a cold install) no longer accumulates the union of every module's AST closure. Each file's working set (hub, type-evaluator caches, dependency map, and diagnostics) is released once its bytecode is captured, with the `.jir` cache as the durable cross-unit channel. A full cold precompile of `jaclang` drops peak memory roughly 9x (about 8.5 GB down to about 1.0 GB). Eviction is scoped to that throwaway compile driver and never runs in the live runtime import path. Set `JAC_NO_UNIT_EVICT=1` to opt out.
- **Refactor: unify precompile drivers behind a shared helper**: the bytecode precompile CLI and the scale binary injector now share a single `precompile_unit` helper instead of each carrying its own copy of the compile, sentinel-patch, and `.jir` write sequence. The per-unit AST closure release that bounds precompile memory now applies to both drivers (previously only the CLI had it).
- **Internal: webhook tests reuse a single MongoDB container**: The webhook test suite now shares one MongoDB testcontainer across its MongoDB-backed cases, dropping the database between tests for isolation, instead of starting and stopping a fresh container per test.

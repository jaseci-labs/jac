- **Fix: native backend resolves `glob`/`enum` declared in a `.na.impl.jac` impl annex** (issue #6754): a `glob` (or `enum`) declared in a native impl-annex file rather than the head `.na.jac` decl file failed to lower with `error[E5090]: Native pathway does not yet support expression 'Name'` (or `'BinaryExpr'`), even though the same declaration in the decl file worked and the `@sv` pathway resolved annex globals correctly. The native pathway now recognizes `*.na.impl.jac` as native (so the annex is coerced to native context and type-checked, giving its declarations a stamped type) and folds impl-annex statements into native global/enum registration (annex declarations live in `impl_mod` sub-modules, not the head module body). Modules without an impl annex are unaffected.
- **Internal: unified native-context detection across the compiler** (issue #6754): "is this module native?" is now a single `Module.is_native` / `is_native_file` predicate, and `in_native_context` is symmetric with `in_client_context` (a whole-native file `.na.jac` / `.na.impl.jac` is native at the module level, not just inside an explicit `na {}` block). The inference gate, the `TypeCheckPass` emit-full gate, the type-evaluator native check, and boundary analysis all read that one predicate instead of the per-site `endswith('.na.jac')` checks that had diverged and hidden the annex from native type inference. Top-level declarations of a whole-native file are now classified native consistently (the type-annotation and `na`-vs-`sv` checks previously mislabeled them).
- **Internal: single codegen body fold shared by the `@sv`, `@cl`, and `@na` backends** (issue #6754): the per-backend module-body merges (`BaseAstGenPass._merge_module_bodies` for `@sv`/`@cl`, and a native-only annex walk) collapse into one `ModuleCodegenPass.collect_codegen_bodies`, so all three backends weave impl annexes (and variant/test bodies, where applicable) identically.

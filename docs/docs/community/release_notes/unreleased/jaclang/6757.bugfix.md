- **Fix: native backend resolves `glob`/`enum` declared in a `.na.impl.jac` impl annex** (issue #6754): a `glob` (or `enum`) declared in a native impl-annex file rather than the head `.na.jac` decl file failed to lower with `error[E5090]: Native pathway does not yet support expression 'Name'` (or `'BinaryExpr'`), even though the same declaration in the decl file worked and the `@sv` pathway resolved annex globals correctly. The native pathway now recognizes `*.na.impl.jac` as native (so the annex is coerced to native context and type-checked, giving its declarations a stamped type) and folds impl-annex statements into native global/enum registration (annex declarations live in `impl_mod` sub-modules, not the head module body). Modules without an impl annex are unaffected.

- **Native: `str()` on an `any`-boxed scalar now returns the value, not the type tag**: In the native backend, calling `str()` on an `any`-boxed scalar (e.g. a value read from a `dict[str, any]`) emitted the `JacVal` discriminant tag byte instead of the value. `str()` now unboxes the `JacVal` at runtime and renders str/int/float/bool/None correctly. The merged result is marked owned (mirroring `_emit_obj_str`), and the str arm copies the boxed string into an owned `rc_alloc` buffer, so the scratch buffers on the int/float/str arms are released rather than leaked and the str arm no longer hands back a borrow that dangles once the owning container is freed.

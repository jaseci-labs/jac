- **Fix: `jac ai --tui` now reaches the plugin agent (hook wiring)**: `JacCmd.run_ai_agent` was declared as a pluggable hook (its docstring says a plugin can register its own implementation to replace the built-in agent) but was never marked `@hookable`, so the `#6856` `@hookable` sweep skipped it. `jac ai` therefore called the built-in byLLM REPL directly and never consulted plugin implementations, which made `--tui` -- and any plugin-registered agent override -- silently ignored. Marking the method `@hookable` routes dispatch through the plugin manager: the built-in `--tui` launcher now runs for `--tui`, while plain invocations fall through to the default REPL. (jaseci-labs/jaseci#6355)
- **Fix: Capture stray stdout/stderr during in-process TUI sessions**: While `jac ai --tui` runs in-process, fds 1/2 redirect to `JAC_AI_TUI_DEBUG_LOG` (or `/dev/null`) so stray prints and tracebacks are logged for post-mortem inspection instead of corrupting the alt-screen renderer; cleanup paths now log exceptions via `_debug_log` instead of swallowing them. (jaseci-labs/jaseci#6355)
- **Fix: surface masked type-resolution failures in the native IR pass**: `NaIRGenPass._resolve_jac_type` triggered on-demand type evaluation (`get_type_of_expression`) inside a bare `except Exception { }`, so any underlying type-inference error was silently discarded and the pass fell through to its primitive/struct fallbacks with no trace of why. The handler now binds the exception and emits a `DEBUG`-level record (with `exc_info`) on the `jaclang.native` logger, so masked type-resolution failures become diagnosable while fallback behavior is unchanged. (jaseci-labs/jaseci#6355)
- **Fix: correct the `JacServe.create_server` return type**: The method constructs and returns a `ThreadingHTTPServer` (each request served on its own thread) but was annotated as `HTTPServer`, the single-threaded base class, so static typing and IDE tooling misreported the serve runtime as non-concurrent. The annotation now reflects the actual `ThreadingHTTPServer` return. (jaseci-labs/jaseci#6355)

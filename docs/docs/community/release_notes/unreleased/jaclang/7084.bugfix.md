- **Fix: `--scale` deploys no longer crash-loop on missing `pymongo`**: the scale runtime's user manager persists to MongoDB by default, but the pod's capability closure only pulled the mongo driver when a `[plugins.scale.database]` backend was spelled out. A scale app that served without declaring one shipped pods that booted, ran `jac install`, then crash-looped at first request with `pymongo required`. A scale-enabled serve now defaults `data.mongo` into its runtime closure (so `jac install` - host or in-pod - installs pymongo), unless the project names a different `[database]` backend.

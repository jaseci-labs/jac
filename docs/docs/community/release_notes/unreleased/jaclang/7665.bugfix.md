- **Fix: byLLM tool-call failures no longer flood logs with ERROR tracebacks**: `Tool.__call__` catches a failing tool call and hands it back to the model as a recoverable `Tool error: ...` observation, but it also logged every occurrence with `logger.exception` (ERROR level plus a full traceback, once per call, with no dedup or rate limit). A confused model repeating the same malformed tool call turned one handled, expected control-flow outcome into an unbounded ERROR-traceback flood (52 identical tracebacks in ~3 min on a single pod under load), burying genuine errors. The handled failure is now logged once at WARNING, message only, naming the tool; the observation returned to the model is unchanged.
- **Fix: a tool call cut off mid-arguments no longer invokes the tool with its arguments silently erased**: the unparseable arguments were replaced with `"{}"` and truncation was only reported when `finish_reason == "length"`, which streaming leaves empty exactly when a call was cut short, so the model was told it had omitted required arguments instead of being asked to retry with a smaller payload.

- **Fix: Type checker now correctly narrows types inside `if` statement branches**: Accessing an attribute or passing a value to a function inside a guarded `if` block no longer produces a false type error. For example, `o.child.value` inside `if o.child { ... }` was incorrectly flagged as an error even though the check guaranteed `o.child` was not `None` at that point. This now works the same way as ternary expression narrowing.

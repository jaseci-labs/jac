- **Fix: `SqliteMemory` `database is locked` under xdist write contention (#7056)**: `SqliteMemory._ensure_connection` set `journal_mode=WAL` + `synchronous=NORMAL` but never set `PRAGMA busy_timeout`, so a contending writer raised `sqlite3.OperationalError: database is locked` immediately (0ms) instead of retrying. It now sets `PRAGMA busy_timeout=5000`, so contending writers retry for up to 5s before failing -- fixing the whole class of contention failures, not just the flaky `test_gcs_storage.jac` teardown that surfaced it. The Firebase CI step also now runs serial (`JAC_TEST_JOBS=0`), matching its sibling step, since all xdist workers share one relative `shelf_db_path` (`.jac/data/anchor_store.db`).

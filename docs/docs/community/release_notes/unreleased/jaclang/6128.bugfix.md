- **Fix: JIR cache stale hit when source content changes with mtime preserved**: `is_module_cache_valid` now verifies the head source file's CRC32 against the hash stored in the JIR header as a fallback after the mtime fast-path. Previously, mtime was the only invalidation signal, so git operations (checkout, stash-pop) that restore files with timestamps older than the cached JIR could silently serve stale bytecode. The write path (`JirWriter.write_module`) is updated to hash raw file bytes consistently with the read-side check.

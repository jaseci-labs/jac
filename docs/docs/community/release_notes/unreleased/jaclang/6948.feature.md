- **Build: `zig build -Ddev` editable dev binary (linked compiler)**: a build mode for compiler work that ships the `jac` binary **without** a bundled jaclang -- it bakes an absolute path to a compiler source dir into the payload and reroutes `import jaclang` there at startup, from any directory (no `[dev] jaclang_source` `jac.toml` stanza required in scope). Because nothing is bundled, the build skips both the ~100 MB compiler-tree copy and the JIR precompile, so it is much faster and ~65% smaller than a release binary; `scripts/fresh_env.sh` now uses it. `-Ddev` links the build root; `-Djaclang-dir=PATH` links an explicit dir. The binary hard-depends on that source tree (dev-only, not distributable) and -- since the compile schedule imports the native passes -- still needs the LLVMPY_* shim placed in-tree (`zig build fetch-llvm`, which `-Ddev` then compiles and places; the build stops with a clear message if it is missing). Builds on the editable dev loop (#6915).

- **Fix: type narrowing inside a loop now follows values across iterations**: Reading a variable inside a `for` / `while` body only considered its value on entry to the loop, so a variable reassigned each pass could report a false type error. For example walking a linked list with `while cur is not None { cur = cur.next; }` could complain that `cur.next` might be `None`, and a variable narrowed by a guard before the loop (`if x is None { return; }`) could lose that narrowing once read inside the loop. In-loop reads are now resolved by iterating the body until the types settle, so recurrences stay narrowed and guard narrowing carries into the loop. Applies to `for`, `while`, iter-for, nested loops, and JSX slot loops.

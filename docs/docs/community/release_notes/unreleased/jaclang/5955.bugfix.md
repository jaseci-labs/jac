- **Fix: clear diagnostic for `glob` declarations inside a code block**: `glob a = 90;` inside `with entry { ... }` or a function body previously surfaced a misleading `E0002 Missing ';'` anchored after the `glob` keyword. The parser now rejects `KW_GLOBAL` in statement position with the new `E0035`, anchored on the `glob` token, with help pointing at `global name;` for referencing module globals.

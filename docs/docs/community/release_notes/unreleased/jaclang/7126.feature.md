- **Checker: WHATWG browser host APIs modeled for cl code**: `jac check` no longer false-positives on standard browser code in client modules. The JS type environment now models `fetch` → `Response` (`ok`, `status`, `json()`, `text()`, `body`), `Headers`, `ReadableStream`/`ReadableStreamDefaultReader` (`read()` → `{done, value}`), `TextDecoder`/`TextEncoder`, a typed `navigator.clipboard` (`writeText`/`readText`), and the document pointer-lock surface (`pointerLockElement`/`exitPointerLock()`). `await` now unwraps `Promise[T]` to `T` (via `Promise.__await__`), so awaited results no longer cascade E1055/E1001. Dynamic payloads (`resp.json()`, `JSON.parse`) intentionally stay `object` — re-type them at the boundary with an `as` cast. Also fixes two evaluator bugs surfaced by the same QA: every `round(...)` call failed E1054 (typeshed's private `_SupportsRound1/2` protocols were missing from the structural-protocol table), and `{**spread}` dict literals inferred the spread dict as one entry value instead of merging its key/value types.

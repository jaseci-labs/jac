- **Fix: native `any`/`str()` gaps for a pure-Jac stdlib (#6943)**: The native (`na`) backend now handles the produce/consume-`any` operations a pure-Jac `json` needs. `str(None)` returns `"None"` (was a SIGSEGV on a null deref) and `str(True)`/`str(False)` return `"True"`/`"False"` (was `"1"`/`"0"`); an `isinstance`-guarded `any`→`dict` consume that accumulated a string under a loop no longer use-after-frees the accumulator (loop-iteration cleanup no longer releases a variable declared outside the loop body); subscripting (`x["k"]`/`x[i]`), iterating (`for e in x`), and `len()` now work on a runtime-typed `any` holding the canonical `dict[str, any]`/`list[any]` layout; an empty `list[any] = []` literal compiles; `str.strip`/`lstrip`/`rstrip` honor a character-set argument; and an empty string is correctly falsy in a ternary condition.

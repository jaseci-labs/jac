- **Fix: false positive E1031 on classmethod calls returning `Self` (#5688)**: `datetime.now(timezone.utc).isoformat()` and any user-defined `@classmethod ... -> Self` chained access raised "Cannot access attribute X for type Self" because `_collect_type_var_bindings` stripped the implicit `cls` parameter before Self could be inferred from arguments. The bindings dict is now pre-seeded with `Self -> receiver_class.clone_as_instance()` whenever the call's first param will be auto-stripped (bound method or classmethod via attribute access). The staticmethod path (`object.__new__(self.__class__)`) is left alone so its `cls: type[Self]` keeps consuming the first positional argument and binding Self the existing way.

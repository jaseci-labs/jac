- **JSX statement slots**: `{...}` in JSX child position is now a unified slot that accepts either a single expression (the existing `{name}` behavior) or a statement-form body. Inside a statement slot, control flow (`if`/`for`/`while`/`match`/`switch`/`with`/`try`) yields JSX children directly, and a bare `return;` is an early-exit guard. Replaces the comprehension/ternary boilerplate previously needed inside JSX.
- **`unsafe_html` ambient builtin**: `unsafe_html(x)` marks `x` for raw-HTML rendering. The client runtime turns the marker into `dangerouslySetInnerHTML` (jac-client React) or `innerHTML` (bare-serve DOM). Named for greppability at the security review boundary.
- **Dynamic JSX tags**: `<@expr />` resolves its element tag from an expression (an identifier, a dotted access, or a brace-wrapped `<@{expr}>`) to a host-tag string, a component reference, or a `str | type` value.
- **`E20xx`/`W2019` slot-body diagnostics**: a slot rejects value-returning `return`, `return`/`break`/`skip` inside template loops, and warns on keyless JSX in a `while` loop.

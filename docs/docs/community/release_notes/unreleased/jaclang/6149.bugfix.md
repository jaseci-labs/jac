- **Fix: `type(x) is <T>` no longer emits bare identifiers in jac2js (#6149)**: Comparisons like `type(x) is str` and `type(x) is not dict` now lower to a runtime `_jac.poly.type_is(x, "str")` helper instead of referencing undeclared globals (`str`, `dict`) that threw `ReferenceError` in strict ES modules. Routing through the helper (mirroring `_jac.poly.contains`) evaluates the subject exactly once, so a side-effecting subject like `type(items.pop()) is dict` is correct, and keeps the lossy Python-to-JS type mapping in one place. Custom archetype/class types (e.g. `type(x) is Animal`) pass the class constructor to the helper for an exact `x.constructor === Animal` match, mirroring Python's exact-type `type() is` semantics rather than subclass-inclusive `instanceof`. The builtin mappings are JavaScript approximations of Python `type()` checks (e.g. no int/float distinction beyond whole-valued numbers), not byte-for-byte Python identity.

- **Fix: `type(x) is str` / `is not dict` no longer emit bare identifiers in jac2js (#6149)**: Comparisons like `type(x) is str` and `type(x) is not dict` now lower to valid JavaScript `typeof`, `Array.isArray`, and related checks instead of referencing undeclared globals (`str`, `dict`) that threw `ReferenceError` in strict ES modules. Builtin typename mappings use JavaScript runtime semantics (approximations of Python `type()` checks), not byte-for-byte Python identity.

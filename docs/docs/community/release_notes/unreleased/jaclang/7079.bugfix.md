- **Fix: clear error for the `--experimental` bundle PVC on RWO-default clusters**: The shared bundle PVC requests `ReadWriteMany`, but its storage class defaulted to the cluster default StorageClass, which on most EKS clusters (gp3 / `ebs.csi.aws.com`) is `ReadWriteOnce` and cannot provision RWX. The deploy then hung until the bundle-loader pod timed out with an opaque message. `build_bundle_pvc` now fails fast with an actionable error naming the PVC, the access mode, and the fix (set `bundle_storage_class` to a RWX-capable StorageClass such as `efs-sc`).

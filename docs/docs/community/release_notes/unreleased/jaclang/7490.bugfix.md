- **Fix: builtin function used in an imported client module**: a builtin function -- `sorted(x)`, `len(x)`, `min(x)`, `sum(x)`, `enumerate(x)`, ... -- called inside an imported `.cl.jac` module compiled to its bare, undefined name (`sorted(x)`) instead of the `_jac.builtin.*` runtime helper, so it threw `ReferenceError: sorted is not defined` in the browser on first execution while `jac check` stayed silent. The same call in the entry module compiled correctly. Imported client modules are now run through the inference schedule before the client backend generates their JavaScript, so calls carry the classification the backend consumes and builtin functions lower correctly in every client module.

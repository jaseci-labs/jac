- **Fix: `--experimental` bundle keeps scale capability intents**: The bundle sanitizer no longer strips the entire `[plugins.scale.*]` namespace from the pod's `jac.toml`. It now removes only the secrets sub-table, in both supported forms (`[plugins.scale.secrets]` and `[scale.secrets]`), so the capability intents (database, storage, monitoring, tracing, ...) survive and the pod's `jac install` can materialize the config-driven runtime closure instead of crash-looping with `Capability 'data.mongo' is required but not installed`.

- **Fix: region handles now drop after ordinary locals in scope and loop cleanup**: both cleanup paths released locals in declaration order, so a region handle declared before other locals had its arena bulk-freed first, and releasing a later local that referenced a region resident read a freed arena header -- silent corruption when the chunks were untouched, a segfault under allocation reuse (canonical failure: a loop opening a per-iteration region with an entry-node binding declared after the handle). Cleanup is now two-pass -- ordinary locals first, region handles last -- matching the escape checker's guarantee that no region reference outlives the handle's scope.

- **Fix: reassigning a `None`-initialized variable inside an `if`-branch no longer drops its evolved type**: the idiomatic `cache = None; if cache is None { cache = set(); } return cache;` pattern was raising a false `E1002: Cannot return NoneType, expected set[str]` even though Python and Pyright accept it. The CFG narrowing walker's assignment-predecessor handler used a strict subtype check, so the new `set[str]` failed the "assignable to NoneType" test and the post-branch type fell back to NoneType. The walker now mirrors `is_evolving_reassign` in `type_checker_pass` — when the declared type is NoneType (from `x = None` without annotation), any later assignment widens the type, matching Pyright's evolving-locals semantics. Only triggers when the declared type is literally NoneType; all other narrowing paths are unchanged.

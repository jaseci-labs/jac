- **Refactor(c2jac): replace two-hop mapper with single CastBuildPass**: Deletes the `CMapper`/`CUniBuilder`/driver architecture (Python-AST intermediate hop + separate direct-IR layer) and replaces it with one `CastBuildPass` that builds uni IR directly from pycparser C-AST. Moves shared C-front helpers (`c_common`, `c_types`, `c_idioms`, `enum_prepass`, `preprocess`) from `jaclang/compiler/c2jac/` into a new `jaclang/compiler/cfront/` package shared with `cbindgen`. Vendors `pycparser` in-tree at `jaclang/vendor/pycparser` and drops the PyPI `pycparser` dependency so the core install stays dependency-free. Gates `NormalizePass`'s synthesized-token line repair on a new `Module.has_synthesized_tokens` flag so it is a no-op for normally-parsed Jac.

- **Refactor(c2jac): fully vendor the C front-end (pcpp + fake_libc)**: Vendors `pcpp` in-tree at `jaclang/vendor/pcpp` (absolute intra-package imports rewritten to relative so it resolves under the namespaced path) and drops the PyPI `pcpp` dependency from `jac.toml` and the payload bundler, so the c2jac front-end now carries zero third-party runtime dependencies. Adds pycparser's `utils/fake_libc_include/` stub headers (omitted from the PyPI wheel, #6973) so system `#include <...>` headers resolve without a toolchain, and trims the unused pycparser codegen tooling (`c_generator.py`, `_ast_gen.py`, `_c_ast.cfg`).

- **Refactor(c2jac): render the transpile report from emitted diagnostics**: the faithfulness report no longer keeps a parallel `c_diag_records` tuple list on the pass; it derives from the diagnostics the pass actually emitted (the base `Transform`'s `Alert` list), removing a second stringly record format.

- **Fix(formatter): unparse clib `import from` blocks instead of crashing**: `NormalizePass.enter_import` now rebuilds a foreign-import's body from `clib_decls` (it previously reconstructed kids from `items` only, dropping the declarations), and `DocIRGenPass.exit_import` guards the empty-block case. Previously `unparse()` / `jac format` raised `IndexError` on any `import from "lib" { def ...; }` clib block, which also blocked feeding synthesized bindings back through the formatter.

- **Refactor(cbindgen): build `Import.clib_decls` IR instead of concatenating text**: `jac cbindgen` now constructs real uni nodes (decl-only abilities for functions, `obj` archetypes for structs, module globals for integer constants) in a new `cfront/cbind_ir.jac` and unparses them, replacing the hand-built `.na.jac` string blocks. Building IR means the emitter can only produce grammatically valid, compilable Jac -- the bindgen tests now push every fixture's output through `nacompile` rather than pinning exact strings. This also fixes two latent defects the string path hid: a variadic's `*rest` param was emitted without the type annotation the checker requires, and extern file-scope variables were emitted as `import from lib { glob x; }` (which does not parse and which native FFI never consumed) -- extern-variable bindings are dropped, since `clib_decls` has no representable form for them.

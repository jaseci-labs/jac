- **JIR container-format constants single-sourced**: the JIR on-disk format constants (`FORMAT_VERSION`, `HEADER_SIZE`, `SECTIONS_MAGIC`, the `SEC_*` section ids, `PRECOMPILE_SENTINEL`, ...) were hand-declared in both `jir.jac` (the writer) and `sealed.py` (the pure-Python section reader that must import before any `.jac` module loads), kept in sync only by a runtime drift-check in `precompile_bytecode.jac`. They now live in one dependency-free leaf, `jaclang/jac0core/jir_format.py`, which both import from; `jir.jac` re-exports them so downstream imports are unaffected. The obsolete drift-check is removed (runtime version skew is still caught by the sealed manifest's `jir_format_version` gate). No behavior change.

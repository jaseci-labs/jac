- **`E1054` now explains why the call did not match**: a failed overloaded call reported only "No matching overload found", leaving you to guess which argument was wrong. It now names the argument and the reason. A callback rejected for its return type reads `the callable passed for parameter 'key' returns 'dict', but must return 'SupportsDunderLT[Any] | SupportsDunderGT[Any]' (overload 2)`; one rejected for its shape reads `takes 2 parameter(s), but a callable taking 1 is expected`; and a rejected value reports the type the overloads expected instead. Sorting records by a field with an inline `key=lambda` is the common case this affects. The hint for unannotated lambdas also stops suggesting the old `lambda d: <type> : ...` syntax, which no longer parses.

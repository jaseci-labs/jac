- **Feature: Exported symbols for native executables**: `jac nacompile` can now export defined symbols into an executable's `.dynsym` so they resolve via `dlsym`/`RTLD_NEXT` (e.g. exporting `close()` for the CEF zygote), and appends `libc.so.6`/`libm.so.6` last in `DT_NEEDED` so libcef's `.init_array` `dlsym(RTLD_NEXT, "close")` resolves correctly. (jaseci-labs/jaseci#6572)
- **Feature: CEF desktop target (`cef`)**: New `CefDesktopTarget` provides a Chromium Embedded Framework renderer alongside the OS-native webview. Set `engine = "cef"` under `[plugins.desktop]` and build with `jac build --client cef`. The loopback server and OAuth broker are shared with the native target; CEF integration is Jac-native (`cef.na.jac` + `libcef_dispatch.so` + `cef-subprocess`) with a pinned CEF 119.x binary fetch and an ABI guard for the verified `libcef.so` version. (jaseci-labs/jaseci#6572)
- **Feature: CEF smoke-test example**: New `notes-app` example exercises the CEF desktop target end-to-end (`jac build --client cef`) and includes desktop bridge, loopback broker, and localStorage diagnostics. (jaseci-labs/jaseci#6572)

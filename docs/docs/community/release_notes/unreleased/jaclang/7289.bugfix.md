- **Fix: `jac start --scale` no longer defaults to the shared `default` namespace and app `jaseci`** (issue #7248): with no `app_name`/`namespace` configured under `[plugins.scale.kubernetes]`, an unconfigured deploy silently landed in the cluster's shared `default` namespace as app `jaseci`, colliding with unrelated workloads there, and a later `jac destroy` of app `jaseci` in `default` prompted to delete the entire namespace. `app_name`/`namespace` now default to a Kubernetes-safe slug of `[project].name` instead (falling back to the legacy `jaseci`/`default` when no project name is usable), so every project gets its own namespace by default; explicit config still overrides.

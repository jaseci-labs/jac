- **Eager schema migration + hook retirement (`jac db migrate` / `schema census` / `schema retire`)**: The declarative `__jac_schema__` drift layer can now be driven to completion instead of accumulating rules forever. `jac db schema census` counts how many live documents each rule still applies to (the proof gate); `jac db migrate` eagerly converges every stored document to the canonical shape using the same repair the runtime applies on read, stripping dual-written alias shadows (only changed rows are rewritten, and a second run is a no-op; `--dry-run` previews). Once an archetype's census reads zero, `jac db schema retire <Class>` removes the dead hook from `.jac` source — both the `static def __jac_schema__` declaration and the `impl <Class>.__jac_schema__` block — refusing unless the census is zero so it can never delete a rule a document still needs (preview by default, `--write` to apply). Backed by a new `census_doc` primitive in `schema_rules` (a single source of truth shared with the repair engine) and a brace-aware source codemod in `schema_rules`/`schema_retire`. This makes a `__jac_schema__` hook *temporary*: declare the drift, let lazy repair carry old data forward, then migrate + census + retire to clear it.

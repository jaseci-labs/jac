- **Fix: walker writes did not refresh cached reads in fullstack apps (#7706)**: in a fullstack SPA, an `sv`-imported walker (a write) failed to invalidate the client's cached reader responses, so the UI kept showing stale data until the 60s cache TTL expired or the page was fully refreshed. For example, liking a post or posting a reply saved on the server but the timeline still showed the old value. Walker writes now correctly clear the reader caches they affect, so the UI updates immediately.

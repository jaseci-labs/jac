- **Fix: `super()` in a method now type-checks as the parent instance**: `super()` inside an archetype method was falling through to the generic "call an instance" path because `_get_type_of_super` returned the parent instance for the bare `super` Name. With no `__call__` match it produced `Unknown`, and `super().method()` cascaded into spurious E1002/E1001/W1051 diagnostics. `validate_call_args` now short-circuits when the call target is `super` — the return type of `super()` is the same as bare `super`, matching Python semantics where attribute access dispatches through the parent's MRO.

- **Fix: decorated class inside a function body now parsed correctly**: When a class declaration with a decorator (e.g. `@dataclass`) appeared inside a function body, the parser routed all decorated items to `parse_ability()` without looking at what the decorator was applied to. This caused the type checker to return `NoneType` for the constructor and report attribute errors on every field access. The fix makes the code-block statement parser call `_dispatch_decorated()` - the same lookahead helper already used at module level - so decorated classes produce an `Archetype` node and decorated functions produce an `Ability` node as expected.

- **Fix: LSP completions offered on all triggers, not just member access**: the langserve completion request only fired on the dot-access operator, so everyday typing (`print`, keywords, `self`) looked broken in clients like jac.nvim and Zed despite a healthy LSP client. It now resolves the enclosing scope when there is no DOT, treats `SpecialVarRef` like `Name`, flushes pending typechecks before collecting candidates, and advertises the real trigger characters (identifier, keyword, blank line) in the server capabilities, so completions appear on every keystroke.

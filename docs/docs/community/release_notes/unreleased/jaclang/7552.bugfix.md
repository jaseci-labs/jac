- **Fix: subscript augmented assignment miscompiled as an RHS store on native**: the subscript branch of native assignment lowering never consulted the augmented operator, so every `seq[i] op= rhs` compiled as `seq[i] = rhs` -- the read-modify step was silently dropped for lists and dicts, all operators, and all receiver kinds (plain locals, owned objects, `&mut` borrows), on the default gc path as well as nogc. Field targets had the same silent RHS-store fallback for float `//=` / `%=`, and float `//=` through any augmented path skipped the `floor()` the plain `//` operator emits. A shared `_apply_scalar_aug_op` now delegates every augmented form to the canonical binary-operator emitter (overflow checks, zero-division raises, floored float `//`, int pow), reading the current element off the already-computed target and index so the index expression is evaluated exactly once; `/=` on int targets and augmented ops on struct-array elements now fail loudly (E5092) instead of storing the RHS. A `py_subscript_aug.na.jac` fixture pins list/dict/field forms and all operators in the native py-semantics suite.

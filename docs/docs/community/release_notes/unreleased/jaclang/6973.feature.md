- **Feat(c2jac): C-to-Jac transpiler with honest tiered lowering**: Implements `jac c2jac <file.i>` - a transpiler from preprocessed C to Jac, reusing the existing `PyastBuildPass` (pycparser C-AST -> CPython `ast` -> uni IR) with a direct-IR path for constructs with no Python-AST analog (C `for`, enums). Every node is routed on two axes - **Mechanism** (`PY_AST` vs `DIRECT_IR`) x **Tier** (Tier-A faithful vs Tier-B best-effort) - and every best-effort (Tier-B) site is flagged inline in the emitted Jac (a `# c2jac:` header summary plus per-site notes) for full transparency. Coverage includes switch/do-while/ternary lowering, faithful for-body lowering, and §7 Tier-A pointer idioms (list-of-objects, ptr-as-ref, realloc-elide).
- **Feat(c2jac): pcpp preprocessing, inline Tier-B reporting, storage-class/qualifier classification**: Adds pcpp-driven preprocessing with CLI flag surface (`-I`, `-D`, `--force-include`, etc.) so `c2jac` consumes real `.c`/`.h` input, and lazy-imports pcpp so the pycparser-only `transpile_c_str` API and its tests run without it. Drops the sidecar JSON report in favor of an inline Tier-B record emitted directly in the Jac output, and classifies C storage-class specifiers and type qualifiers (`const`/`static`/`extern`/`volatile`/`register`/`auto`) into faithful-vs-honest lowering rows. Mapper gains last-write-wins list-bound tracking with deduplicated `flatten_append`, and the compat-matrix test suite gains coverage foundations across P1-P4 phases.

- **Feat(c2jac): direct-lift C-to-Jac transpiler via CastBuildPass**: Implements `jac c2jac <file>` using a single `CastBuildPass` (`Transform[CModuleAst, Module]`) that lifts pycparser C-AST nodes directly into Jac uni IR with no intermediate Python-AST hop. Faithful constructs lower transparently; unfaithful ones emit inline `# c2jac:` diagnostic comments (E/W42xx series) and placeholders so output is always parseable. Coverage includes functions, structs, enums (faithful `enum` or frozen `glob` batch per shape analysis), arrays, control flow (for-promotion to `IterForStmt`, while, do-while, switch to if-elif chain), pointer idioms (list-of-objects, ptr-as-ref, realloc-elide), and ternary/compound-assignment expressions.
- **Feat(c2jac): pcpp preprocessing and C front-end split into cfront/**: Adds pcpp-driven preprocessing with full CLI flag surface (`-I`, `-D`, `-U`, `--force-include`, `--nostdinc`) so `c2jac` consumes real `.c`/`.h` input without a system toolchain. Shared C-front helpers (type classification, idiom matchers, enum prepass) live in `jaclang/compiler/cfront/` and are reused by both `c2jac` and `cbindgen`.

- **Feat(c2jac): C is a first-class compiler input**: `.c`/`.h`/`.i` files handed to the compiler are now dispatched to `CastBuildPass` by `parse_str` (mirroring the `.py` -> `PyastBuildPass` path) instead of falling through to the Jac reader. `.c`/`.h` are preprocessed with the vendored fake-libc include path (`.i` is taken as already-preprocessed) and lifted straight to uni IR, so C is a source language rather than only a `jac c2jac` CLI transpile.

- **Feat(c2jac): import-time C header binding via `clib(...)`**: `import from clib("z", header="zlib.h") { compress, uncompress }` reads the named header at load time and synthesizes `Import.clib_decls` in place from the selected symbols, so header-driven FFI bindings flow through the existing native substrate (`BoundaryAnalysisPass` -> `targets/foreign.jac` -> `clib_abi`) with no intermediate `.na.jac` file to generate and import. The synthesis reuses the same header-extraction core as `jac cbindgen`; the compact `clib(...)` form round-trips through `unparse` instead of expanding to the full binding.

- **Feat(c2jac): function quarantine + multi-TU project provenance**: Behavior/hole fidelity sites (`W4205`/`W4206`/…/`W4290`) now quarantine their containing function as a call-time `raise` trap (style sites like `W4201` stay warnings only), so Tier-B no longer leaves mid-body `__c_unsupported__` landmines. `jac tool c2jac -o` writes a machine-readable `*.c2jac.report.json` sidecar alongside the lifted Jac; `--project <dir> -o <outdir>` lifts every `.c` with a cross-TU symbol registry, wires `import from <stem> { … }` for foreign calls, and emits an aggregate `project.c2jac.report.json`.
- **Feat(c2jac): LP64 porting libc + shared scalar map**: Preprocess prefers `compiler/cfront/porting/` (LP64-correct `stddef`/`stdint`/`limits` and `jacport.h`) ahead of pycparser fake libc. Bindgen and lift share `lp64_scalars.jac` so `size_t` / fixed-width / `Py_ssize_t` resolve by name to the same widths.
- **Feat(c2jac): lift behavioral oracle is bytecode vs original C**: `test_c2jac_lift_oracle` compares `cc` of the original `.c` against `jac run` of the lifted Jac (including idiomatic `list[...]` lifts jac2c cannot re-emit). C-shaped POD lifts additionally round-trip through jac2c+`cc` in `test_jac2c_roundtrip`.
- **Feat(c2jac): differential oracle gate with timeout + trap regime**: `lift_oracle` subprocesses time out (hang → divergence), auto-derive MATCH vs TRAP from `quarantined_functions`, and collect-all mismatches. W4208 (continue-in-for) moves STYLE→BEHAVIOR so under-quarantining cannot ship a hang as "faithful".
- **Fix(c2jac): runnable globals + defaulted struct fields**: Module-scope C decls lift to BSS-zeroed `glob`s; functions that assign them emit `global name;`. Struct `has` fields get zero defaults so malloc→`Type()` and `[Type() for _ in range(n)]` construct without args.
- **Fix(c2jac): continue-safe for lowering (Option A)**: `_for_promotable` accepts `>`/`>=` so descending fors become IterForStmt (Jac runs count_by on continue). Non-promotable for→while lowerings splice the step before each loop-level `continue`, so W4208 no longer fires for faithful continue-in-for.
- **Fix(c2jac): C integer `/` lowers to Jac `//`**: When both operands are known ints, `/` and `/=` become `//` / `//=` so truncating C arithmetic matches under the bytecode oracle (unlocks `for_cast.c` MATCH despite STYLE W4201 cast elision).
- **Feat(c2jac): oracle covers residual W4208 + behavior TRAPs**: Unmappable-cond W4208 fixtures and other quarantined callables (`do_while_continue`, `switch_fallthrough`, nested dirty/clean, edge-case holes, …) sit in the differential gate; STYLE skips shrink to fixtures without a stable differential entry.
- **Feat(c2jac): char*/str cursor arithmetic**: Plain/`signed`/`unsigned` `char*` lift as `str` cursors - `*p` → `ord(p[i]) if p[i:] else 0`, `p++`/`p+=1` → `p = p[1:]`, `p+n` → `p[n:]`. Scalar `char` and character constants are ints (C promotion). Unlocks `cpython/pystrcmp.c` MATCH and turns former `*(p+1)` holes into faithful loops.
- **Fix(c2jac): hello oracle avoids `main` link collision**: `lift_oracle.run_c` compiles fixtures with `-Dmain=__c2jac_fixture_main` so a driver-supplied `main` can call into fixtures that define their own (e.g. `hello.c`).
- **Feat(c2jac): oracle covers remaining STYLE breadth fixtures**: `obj_array.c` writes before read (no UB vs zero-default diverge); `struct_shape.c` gains `point_sum`; `cpython/rotatingtree.c` exposes deterministic `RotatingTree_RandomBits` (seeded LCG). `_ORACLE_SKIP` is empty.
- **Fix(c2jac): zero-default uninit locals**: Bare locals (`Point p;` / `int x;`) now emit `= Point()` / `= 0` the same way file-scope BSS and `has` fields already did, so field stores don't hit "local not associated with a value".
- **Fix(c2jac): struct compound inits → ctor KWPairs**: `Point p = {3, 4, 0}` and `.y=4,.x=3` lower to `Point(x=3, y=4, z=0)` (field-order KWPairs), including nested struct fields and arrays of structs. Nested designators / array designators still STYLE `W4209`.
- **Feat(jac2c): hash-stable type ids**: Ref-archetype `type_ids` use name-derived `stable_type_id` (FNV via `stable_osp_tag`, reserved floor 2). `exception_type_id` hashes all names except `Exception=0`, so registry order no longer renumbers the ABI.
- **Fix(jac2c): `@__jac_value__` POD ctors compile and run**: Value-arch `Type(...)` calls emit C compound literals (KWPair/positional/defaults), constructor calls type via `field_types`, and ability bodies only wrap a cleanup frame when needed (no orphan `jac_exc_pop`). Minilib lift→jac2c→`cc` is now a behavioral oracle against the original `.c`.

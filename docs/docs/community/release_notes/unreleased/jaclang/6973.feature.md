- **Feat(c2jac): direct-lift C-to-Jac transpiler via CastBuildPass**: Implements `jac c2jac <file>` using a single `CastBuildPass` (`Transform[CModuleAst, Module]`) that lifts pycparser C-AST nodes directly into Jac uni IR with no intermediate Python-AST hop. Faithful constructs lower transparently; unfaithful ones emit inline `# c2jac:` diagnostic comments (E/W42xx series) and placeholders so output is always parseable. Coverage includes functions, structs, enums (faithful `enum` or frozen `glob` batch per shape analysis), arrays, control flow (for-promotion to `IterForStmt`, while, do-while, switch to if-elif chain), pointer idioms (list-of-objects, ptr-as-ref, realloc-elide), and ternary/compound-assignment expressions.
- **Feat(c2jac): pcpp preprocessing and C front-end split into cfront/**: Adds pcpp-driven preprocessing with full CLI flag surface (`-I`, `-D`, `-U`, `--force-include`, `--nostdinc`) so `c2jac` consumes real `.c`/`.h` input without a system toolchain. Shared C-front helpers (type classification, idiom matchers, enum prepass) live in `jaclang/compiler/cfront/` and are reused by both `c2jac` and `cbindgen`.

- **Feat(c2jac): C is a first-class compiler input**: `.c`/`.h`/`.i` files handed to the compiler are now dispatched to `CastBuildPass` by `parse_str` (mirroring the `.py` -> `PyastBuildPass` path) instead of falling through to the Jac reader. `.c`/`.h` are preprocessed with the vendored fake-libc include path (`.i` is taken as already-preprocessed) and lifted straight to uni IR, so C is a source language rather than only a `jac c2jac` CLI transpile.

- **Feat(c2jac): import-time C header binding via `clib(...)`**: `import from clib("z", header="zlib.h") { compress, uncompress }` reads the named header at load time and synthesizes `Import.clib_decls` in place from the selected symbols, so header-driven FFI bindings flow through the existing native substrate (`BoundaryAnalysisPass` -> `targets/foreign.jac` -> `clib_abi`) with no intermediate `.na.jac` file to generate and import. The synthesis reuses the same header-extraction core as `jac cbindgen`; the compact `clib(...)` form round-trips through `unparse` instead of expanding to the full binding.

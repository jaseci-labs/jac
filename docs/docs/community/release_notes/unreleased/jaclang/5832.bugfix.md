- **Fix: JSX block comments (`{#*...*#}`) now parse and format correctly**: Placing a block comment inside a JSX element body previously raised E0004/E0002 parse errors because the lexer strips block comments before the parser runs, leaving a bare `{}` the parser could not handle. The parser now detects the comment slot, the formatter wraps the comment in `{}` when emitting it inside JSX, and round-trip formatting is stable.

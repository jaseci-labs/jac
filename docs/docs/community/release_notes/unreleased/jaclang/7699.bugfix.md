- **Fix: byLLM no longer forces a `temperature` on every request (#7687)**: byLLM injected a default `temperature` (0.7) into every model call even when the caller never set one. This broke GPT-5 / o-series models and Claude extended-thinking mode (which only accept `temperature=1`), and slipped past `litellm.drop_params` when routed through aggregators like OpenRouter, surfacing as `OpenAIException - Unsupported value: 'temperature' does not support 0.2 with this model. Only the default (1) value is supported.`. `temperature` is now sent only when explicitly set via `by llm(temperature=...)` or `[byllm.call_params]` in `jac.toml`; otherwise it is omitted so the provider applies its own default. Note: calls that relied on the implicit `0.7` will now use the provider default (often `1.0`) unless a temperature is set explicitly.

- **`x in any_value` no longer raises false E1111 "does not support **contains**"**: writing `"key" in dict_or_any` where the right-hand side was typed `any` was being rejected even though Any supports all operations per PEP 484 (and both mypy and Pyright accept it). The `in`/`not in` validator now correctly recognizes the prefetched `typing.Any` class.

- **Fix: `jac nacompile` records the real versioned soname for a bare clib import**: A bare logical C-library import (e.g. `import from curl { ... }` / `import from z { ... }`) emitted the *unversioned* `lib<name>.so` into the binary's `DT_NEEDED`. That bare name only exists when the library's `-dev` package is installed; on a stock host only the runtime soname `lib<name>.so.N` is present, so the produced binary failed to load (`error while loading shared libraries: libcurl.so: cannot open shared object file`) -- a module would "work" only by the accident of a dev symlink being present (e.g. `libz.so`) and break otherwise (e.g. `libcurl.so`). `nacompile` now resolves a bare soname to the library's real soname (via `ctypes.util.find_library`), exactly as libc/libm are already recorded as `libc.so.6` / `libm.so.6`; unresolved names fall back to the bare soname unchanged. This is the foundation step toward self-contained native binaries (jaseci-labs/jaseci#6976).

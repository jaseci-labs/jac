- **Fix: Gateway pod client-build fallback no longer runs the whole-project type-check gate**: The in-pod fallback now builds with `--client web` instead of a bare `jac build`, so a fullstack app with type-check errors no longer deploys green while silently serving a JSON 404 at `/`. The in-pod build is a packaging step, matching the host-side client build; the dev-time `jac build` type-check gate is unchanged.

- **Fix: variables read after a loop keep the types assigned inside the loop**: A variable assigned inside a `for` or `while` body is now correctly typed after the loop as the union of those assignments and its pre-loop value, instead of collapsing back to the pre-loop type. For example, `out` in `fb = None; for x in items { fb = 5 } out = fb` now hovers as `int | None` rather than `None`. Loop-condition and iterable narrowing (such as `while x is not None`) is unchanged.

- **Native: deep release is now O(1)-stack**: Releasing a managed value on the native backend has always been deep — an object's destructor releases every managed field/element and cascades into any that in turn reach refcount zero — but that cascade ran as C-stack recursion, so tearing down a long acyclic graph (a linked list, a deep AST) overflowed the stack on drop (a chain of a few hundred thousand nodes SIGSEGV'd even under the default 8 MB stack). The transitive teardown inside `__rc_release_simple` is now driven by an explicit heap worklist: the first release on an idle call drives the drain, and any release re-entered from a destructor appends its pointer and returns, so the whole subgraph is reclaimed in constant stack space. Reclamation stays deep, full, and synchronous (acyclic finalization order is unchanged). Reclaiming a reference *cycle* still requires `--gc cycles`.

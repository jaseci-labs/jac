- **Fix: `jac check` no longer reports false errors on the native LLVM backend**: The native LLVM IR-gen pass is built on `llvmlite`, which ships no type information, so the type checker previously fell back to its unannotated source and flagged its core API (such as `Value.type`, `PointerType.pointee`, and `Type.as_pointer()`) as untyped. A bundled `llvmlite` type stub now lets the checker resolve this API correctly, so the native pass type-checks cleanly with no change in runtime behavior.

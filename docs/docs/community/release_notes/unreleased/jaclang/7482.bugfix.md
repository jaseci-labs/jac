- **Fix: `jac check` degrades gracefully instead of aborting when the vendored typeshed stubs are missing**: with `builtins.pyi` absent (a checkout that hasn't run `zig build`/`fetch-typeshed`), type inference is skipped, yet the type-check schedule still carried `OwnershipCheckPass` (`REQUIRES=('cfg','inference')`), so the pass scheduler hard-failed with a misleading `RuntimeError: scheduling error`. The scheduler now drops any pass whose declared `REQUIRES` can't be satisfied (iterated to a fixed point so a dropped provider also drops its dependents); `jac check` emits a one-time warning naming the missing typeshed and the `zig build` remedy; and a fatal compile exception is surfaced as a counted error instead of the contradictory "0 errors, 0 warnings".

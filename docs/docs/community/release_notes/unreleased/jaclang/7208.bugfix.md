- **Fix: native cycle collector no longer clobbers GC flag bits when recoloring collected objects**: `collect_white`'s recolor-to-black was the last raw write to the RC color byte, storing `RC_COLOR_BLACK` (= 0) over the whole byte and clearing the buffered/dead/teardown flags by coincidence. It now goes through the same masked `_emit_rc_color_store` path that `mark_gray`, `scan_black`, and `scan` already use, so only the two color bits change. Behavior is unchanged today (affected objects are already white and about to be freed), but the write no longer silently depends on `BLACK == 0`. (#7208)

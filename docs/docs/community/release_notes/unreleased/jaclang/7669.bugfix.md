- **jac-scale gateway deploy now fails loudly when the client build fails (#7669)**: following #7668 (which switched the in-pod build to `jac build "<entry>" --client web`), the gateway pod no longer swallows a genuine client-build failure with `|| echo`. Under the pod's `set -euo pipefail`, a failed build now exits non-zero and crash-loops the gateway, so `_wait_for_fleet_rollout` reports the deploy as failed instead of shipping a headless app that serves a JSON 404 at `/`. Also corrects the `jac build` gate hint to name the real flag, `--no_typecheck`.

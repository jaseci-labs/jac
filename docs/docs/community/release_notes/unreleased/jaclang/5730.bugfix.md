- **Fix: Type checker now recognizes `@dataclass` class fields declared inside a `with entry` block**: When a `@dataclass` class was declared inside a function body and its fields were written inside a `with entry` block, the type checker failed to recognize those fields. The class constructor incorrectly returned `NoneType`, and accessing any field on an instance produced `E1030` attribute errors. Three issues were fixed: (1) local `class` declarations with a `@dataclass` decorator now produce a proper `ClassType` instead of `NoneType`; (2) `_build_field_params` now walks `WithStmt` body children to collect fields from `with entry` blocks; (3) `lookup_member_symbol` now searches `WithStmt` nested scopes when the symbol table is a local class. A bonus fix prevents `_build_field_params` from scanning built-in base classes like `object`, which was picking up `__doc__`, `__module__`, and `__annotations__` as fake constructor parameters.

- **Refactor: Visitor double-dispatch for compiler passes (native-lowerable)**: The reflective `enter_node`/`exit_node` dispatch in `UniPass`, which selected each pass ability by formatting the runtime class name into `hasattr`/`getattr(self, f"enter_{snake(type(nd).__name__)}")`, is replaced with classic visitor double-dispatch. `enter_node`/`exit_node` now call `nd.accept_enter(self)`/`nd.accept_exit(self)`; each handled node kind has a generated `accept_<verb>` override that calls the pass's `enter_<kind>`/`exit_<kind>` ability directly (a direct, exact-type, O(1) double-vtable call), and unhandled kinds fall through `UniNode`'s base accept to the pass's overridable `default_<verb>` hook. `UniPass` carries a generated `enter_<kind>`/`exit_<kind>` base surface that routes to `default_<verb>`, so a pass ability simply overrides its slot. Run `jac gen-uni-dispatch [--verify]` to (re)generate the `unitree.impl/dispatch.impl.jac` and `uni_pass.impl/surface.impl.jac` annexes. This removes the string-keyed reflection the native backend could not lower (a self-hosting keystone), behaves identically to the old dispatch across every pass and node kind, and drops the per-node name formatting from the CPython build's hot path.

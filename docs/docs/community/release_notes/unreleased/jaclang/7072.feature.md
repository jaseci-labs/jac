- **Feature: musl/Alpine Linux binary (Python backend)**: `jac` now publishes a `jac-<ver>-linux-<arch>-musl` release binary, built natively against musl, so it runs on Alpine and other musl distros where the glibc binary fails with `GLIBC_… not found`. It ships without the LLVM native (`na`) backend -- there is no musl LLVM release to statically link the `LLVMPY_*` shim against -- so `jac run`/`check`/`test` run on the Python backend while `jac nacompile`/native AOT report a clear "native backend not available" message instead of crashing. To support this, the codegen schedule now probes for the LLVM shim (`shim_available()`) and omits the native passes when it is absent rather than faulting in `NativeCompilePass.postinit`, and `install.sh` detects a musl host (via `ldd`/the musl loader) and fetches the matching asset.

- **Fix: False type errors on `bytearray` slice assignment** (issue #6759): Writing `buf[0:4] = b"..."` incorrectly reported E1001 errors. The type checker was using the read type of `buf[0:4]` (bytearray) to validate the write, but writing to a slice uses `__setitem__` which accepts `bytes`. Now correctly skips the assignment check for slice targets, fixing 190 false positives across the native linker files.

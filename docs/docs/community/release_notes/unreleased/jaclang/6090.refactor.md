- **Refactor: FieldPredicate as typed backend-neutral predicate protocol**: The compiler now emits typed `FieldPredicate(op, value)` objects (with plain operator names: `lt`, `lte`, `gt`, `gte`, `eq`, `ne`, `in`, `nin`) instead of MongoDB-flavored dicts for `[?:Type, ...]` filters. `QueryPlan.field_predicates` is typed as `dict[str, list[FieldPredicate]]`. Backends translate to their native syntax at query time; `FieldPredicate` lives in `jac0core.filter_spec` to avoid circular imports.
- **Refactor: every typed filter carries a `FilterSpec`**: `[?:Type, ...]` always lowers to a `FilterSpec` carrier (predicates ride along when pushable, otherwise empty), so the planner reads the archetype type and field predicates structurally instead of inspecting the filter lambda's bytecode. Predicate extraction now covers the field in either operand order (`28 > age`), chained comparisons (`20 <= age < 50`), `in`/`not in` against literal collections, and negative literals (`age < -1`). All of these previously fell back to a per-node lambda; the constant-on-left and chained forms were additionally dropped from the generated lambda altogether (a correctness bug).

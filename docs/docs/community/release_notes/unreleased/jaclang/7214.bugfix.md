- **Fix: `include`d package members resolve through member access**: accessing a name surfaced into a package only via `include` (absorb) through the package namespace (e.g. `ir.Value`) resolved to `Unknown` when the included submodule imported back from its own package (circular self-import). This poisoned the vendored llvm `ir`/`binding` surface — the native passes reach it via `import from ...llvm { ir }` then `ir.Value` / `ir.Module` / … — cascading ~1,700 downstream `E1030`/`E1032` errors. The member-access path now falls back to the same absorb-aware resolution the `from pkg import X` path already used, and the base llvm `Value` class declares its `type` attribute.

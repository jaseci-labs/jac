- **Fix: the Redis L2 cache now keeps its own memory bounded on any deployment**: when connecting to a Redis it did not provision (managed, docker, or standalone), Jac-Scale checks the server's memory limit on first connect and applies `redis_max_memory` with an eviction policy when none is set, falling back to a clear warning if the server refuses. A `redis_default_ttl` of `0` no longer writes non-expiring keys unless the server is configured to evict them, so cached anchors can never grow without bound. Set `redis_enforce_eviction = false` to opt out (#6885).

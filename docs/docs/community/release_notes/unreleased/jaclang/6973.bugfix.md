- **Fix(c2jac): char-literal ordinals, tight formatting, hyphenated CLI args**: A C char constant combined with a numeric operand (e.g. `i < 'z'`) now lowers to its integer ordinal (`i < 122`) instead of a type-mismatched string comparison, while a char-to-char compare (`c == 'a'`, where `c` is a char->str variable) keeps the readable string form. Front-end-spliced IR no longer emits spurious blank lines: synthesized statement terminators (half-stamped `end_line`) and block closing braces (stamped to the header line) are repaired to real source lines, so emitted Jac is as tight as hand-written. Hyphenated `jac c2jac` CLI flags such as `--force-include` normalize to underscored argument dests (#6115).

- **Fix(c2jac): initializer reads mis-resolving to the constructor**: Reading a pycparser node's `init` field via `node?.init` returned the Jac constructor rather than the C initializer (a `jac format` lint-fix had rewritten a safe `getattr` into `?.init`), so every `Decl`/`For` initializer degraded to an unsupported-construct placeholder. Reads now go through a variable-field `getattr` helper the formatter cannot rewrite, and variadic functions (`...`) are correctly flagged with W4207.

- **Fix(c2jac): clib header binding formats tight and invalidates on header edits**: The compact `import from clib("m", header="hdr.h") { ... }` form now round-trips through `unparse`/`jac format` with tight spacing (`DocIRGenPass.exit_import` rendered the raw import tokens with a space after each, producing loose `clib ( "m", header = "h.h" )`; it now emits the call form tightly, like a `FuncCall`). Separately, a bound C header is now a first-class cache dependency: the `.jir` module key and the mtime fast-path both fold in the header (and its quoted-local `#include` closure) recovered statically from the source, so editing, adding, or removing a bound header re-synthesizes the bindings instead of serving a stale cache.

- **Fix(jac2c): landing-pad ownership so RC composes with setjmp/longjmp**: Exception frames now record owned RC locals (`jac_exc_track` / `jac_exc_untrack`); on throw the runtime walks the frame stack, runs each landing pad, and longjmps only at a catching `try` frame. Functions that own RC locals push a cleanup-only frame so a raise out of a callee still releases that callee's locals. Try-body owned decls are function-hoisted so handlers can null them after unwind, and raise payloads get defined ownership via `jac_exc_throw_owned` / `jac_exc_clear` / `jac_exc_take_payload` / `jac_exc_rethrow`. `OwnershipFactsPass` also walks `try`/`if`/`while`/`for`/`raise` bodies so constructor calls inside them are classified owned (previously defaulted to borrowed and double-retained). Proven by `exc_rc_unwind.jac` with `jac_rc_live() == 0` after try-local and cross-callee raises.

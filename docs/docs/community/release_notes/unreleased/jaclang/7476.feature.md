- **Feature: per-service `deployment_overlay` + profile-aware plugin configs**: `[scale.microservices.services.NAME.deployment_overlay]` merges a raw Deployment fragment into the generated manifest at build time (k8s-style named-list merge, identity/selector fields protected), so nodeSelector, probe tuning, extra volumes/env/init containers land on the FIRST rollout instead of being kubectl-patched into a second one. Plugin configs (including jac-scale's) now apply `JAC_PROFILE` / `default_profile` overlays (`jac.<profile>.toml`) and `jac.local.toml` exactly like the CLI - previously every plugin silently ignored profile files, so per-environment deploy settings (e.g. a 1-replica `jac.preview.toml`) never reached the deployer.

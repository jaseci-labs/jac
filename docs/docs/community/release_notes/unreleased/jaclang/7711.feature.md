- **Feature: per-file diagnostic for `pages/` modules that are not routes**: #7677 warned only when a `pages/` tree produced zero routes and zero layouts, so a partial miss stayed silent - if some pages returned `JsxPage` (or a layout resolved), a sibling that still returned `JsxElement` dropped from the route table with no signal, and that URL simply 404'd on a green build. The route scan now names each unmigrated file after every successful scan (startup and dev-server rescan), reporting the export and the exact return-type fix (`JsxLayout` for `layout.jac`, `JsxPage` otherwise) while never re-flagging files that did resolve. The message stays advisory ("...or leave it if it is a co-located component") because a `pages/` helper returning `JsxElement` on purpose cannot be told apart from a forgotten annotation without import analysis.

- **Feature: Pluggable client entry generators in core**: Core now exposes a target-keyed entry-generator registry so HMR, eject, and the shared client compiler can emit the correct bootstrap script per target (web `createRoot` vs React Native `AppRegistry.registerComponent`) without hard-coding jac-client internals. Built-in web fallbacks keep core-only workflows working when jac-client is not installed; plugin-provided targets such as `react-native` register their generators at jac-client load time.
- **Feature: Forbid raw HTML host tags in mobUI projects (E1105)**: Setting `client_kind = "mobui"` under `[project]` in `jac.toml` marks a project as targeting React Native as well as the web. In a mobUI project, raw HTML intrinsic tags in JSX (`<div>`, `<span>`, `<button>`, ...) now raise `E1105` with a fix-it pointing at the `@jac/ui` / React Native component to use instead (`div` → `View`, `span` → `Text`, `button` → `Pressable`, `img` → `Image`, ...). Uppercase user components are unaffected. Enforcement runs in the always-on IR-generation schedule, so it fires on both `jac check` and `jac build`/dev, including `.cl.jac` client modules, which the type checker deliberately skips.
- **Enhancement: Scope-aware JSX host-tag detection (E1105 / W1050)**: The mobUI-project guard now uses symbol-table scope resolution rather than a lexical case check. Imported lowercase components (`<myWidget>`) are recognised as in-scope user components and are no longer flagged with `E1105`. The type evaluator is updated in parallel so that in-scope lowercase components in mobUI projects also receive correct prop-type validation without spurious `W1050` ("Unknown intrinsic JSX element"). Web-project behaviour is unchanged.

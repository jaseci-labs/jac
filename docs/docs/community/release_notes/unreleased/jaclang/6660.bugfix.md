- **Fix: Multiple `with entry` blocks in a native module (#6660)**: `jac nacompile` on a `.na.jac` module with two or more module-level `with entry { ... }` blocks crashed during IR generation with `llvmlite.ir._utils.DuplicatedNameError: jac_entry`, because the native backend emitted a separate `jac_entry` function per block. The blocks are now concatenated, in source order, into a single `jac_entry` (with the glob-init / `JAC_NO_GC` / imported-module init preamble emitted exactly once), mirroring the Python target. Single-entry modules are unchanged.

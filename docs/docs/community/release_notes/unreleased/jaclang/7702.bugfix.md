- **Fix: deploy status probe falsely fails on TLS-enabled ingresses**: The post-deploy health probe requests `http://<load-balancer-hostname>/<path>` and requires a 200. Once `enable_tls` puts `ssl-redirect` on the app ingress, that request is redirected to https on the same hostname, where certificate verification can never pass (the cert is issued for the app's domain, not the load balancer's name), so every attempt raised `SSLError`, the full retry budget burned (12 minutes), and a healthy rollout was reported as "Application failed to deploy" on every redeploy after enabling HTTPS. The probe now retries the same request without certificate verification when it hits `SSLError`; all other paths are unchanged.

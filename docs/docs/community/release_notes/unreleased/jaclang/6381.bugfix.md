- **Fix: Native pass LLVM type containers typed as `object`**: The native IR-gen pass tracked its LLVM struct types in `object`-typed fields (`struct_types`/`list_types`/`dict_types`/`set_types`/`tuple_types` as `dict[str, object]`, and `complex_struct_type`/`range_struct_type`/`slice_struct_type`/`jacval_type` as `object`), so every lookup resolved to `object` and the `.as_pointer()` / `.set_body()` calls on them were reported as attribute errors. These fields only ever hold `ir.Type` (struct) values, so they are now typed precisely. Pure annotation change with no runtime effect; clears ~110 checker errors across the pass.

- **Fix: `jac start --scale` surfaces init-container crashloops instead of hanging**: The fleet-wait phase only inspected app `container_statuses`, so a crashlooping init container (e.g. `jac-bootstrap` failing with `exec format error`) was invisible and the deploy spun to the 1200s timeout with no explanation. It now scans init and app container statuses, fails fast with the real reason (pod diagnostics plus the container `terminated.message`), scopes polling to the fleet's own pods, and logs only on readiness-state change.

- **Feature: `[dependencies.npm.native]` for the React Native scaffold**: `jac setup react-native` now merges the project's `[dependencies.npm.native]` packages into the generated Expo workspace's `package.json`. The scaffold baseline only covers what every Expo app needs, so an app's own native modules (`react-native-webview`, `react-native-svg`, `@react-native-cookies/cookies`, ...) previously had to be added by hand with `bun add` after every fresh scaffold, and a clone that skipped that step produced a workspace the app could not bundle. The flat `[dependencies.npm]` set stays web/npm-side and is not installed into the Expo workspace, so a monorepo whose web app depends on `monaco-editor` or `react-dom` does not drag those into the mobile build.

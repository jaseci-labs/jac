- **Fix: structured `by llm` returns no longer silently deserialize to all-defaults on Title-Case keys**: when a model keyed a structured object's JSON in Title Case (`"Root Cause"`) instead of the field's snake_case name (`root_cause`) -- which frontier models do stochastically -- byLLM dropped the unmatched key and every field fell back to its default, turning a correct answer into an empty object with no error raised. Deserialization (`json_to_instance` and the finish-tool `parse_arguments` path) now runs a type-directed key repair that maps Title-Case keys back onto the real dataclass fields while leaving the keys of a genuine `dict[str, V]` field untouched, and logs any key that still matches no field instead of dropping it silently.
- **Fix: tool and finish-tool schemas no longer invite the key drift**: `tool_to_schema` now strips the Title-Cased `title` label from nested object fields (previously only top-level params) and emits `strict: True`, mirroring `type_to_schema`, so providers enforce the exact property names rather than letting the model rename fields.

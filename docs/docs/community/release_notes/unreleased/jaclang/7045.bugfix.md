- **Fix: jac-desktop routes walker/function invokes through inprocess dispatch**: The native webview host only wired `_PH_DISPATCH` through `registry.bootstrap_plugins` (`_host.setup()`), so an incoming `window.__jac_invoke` had no path to the in-process walker/function runtime: it either depended on plugin-host setup having run or fell through to the legacy `_host.dispatch` with no way to reach `inprocess_dispatch`. A unified `_jac_unified_dispatch` (exposed as `_dispatch_fn` in the host's `DISPATCH_PY`) now parses the incoming request and, for a `{"kind": "walker" | "function"}` JSON payload (also unwrapping the single-string-list variant), dispatches via `inprocess_dispatch.dispatch`; anything else falls back to `_host.dispatch`. The host boot points `_PH_DISPATCH` at `_dispatch_fn` directly, so invokes route through the unified path without waiting on plugin-host setup.

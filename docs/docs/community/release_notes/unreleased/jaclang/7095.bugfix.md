- **Fix: a variable assigned inside a loop keeps its type after the loop**: Reading a variable after a `for` / `while` loop ignored assignments made in the loop body and fell back to the variable's pre-loop type, so a following guard could not narrow it. For example `fb: int | None = None; for x in xs { fb = 5; } if fb is not None { return fb; }` reported a false `int | None` vs `int` error. The type after a loop is now the union of the pre-loop type and the values assigned in the body, so guards like `is not None` narrow correctly. Applies to `for`, `while`, nested loops, and augmented assignment (`+=`).

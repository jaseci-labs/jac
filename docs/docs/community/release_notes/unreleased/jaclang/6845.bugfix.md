- **Fix: transitive `.na.jac` import type loss on incremental builds (#6793)**: Native-annotation (`.na.jac`) modules are now excluded from the JIR cache fast-path. JIR does not serialize `Expr.type`, so a cache-loaded native tree arrived untyped and cross-module struct types reached transitively were silently dropped to `<Unknown>` on incremental builds, producing spurious `E5090`/`E2018` errors. These dependencies are now always fresh-compiled.

- **Fix: no spurious CFG edge for a comprehension inside a ternary**: `jac tool ir cfg` no longer draws an invalid edge out of the function when a `return` ternary has a list comprehension in its true branch (e.g. `return [x for x in c.items.data] if c.items else [];`). The comprehension splices an `InnerCompr` into the flow for type narrowing, and that node heads no basic block; the dot renderer used to default such unresolved targets to block 0, drawing an edge into the first declaration (or a self-loop on the function). Unresolvable targets are now skipped instead. The narrowing edge in the CFG is unchanged - only its rendering is fixed.

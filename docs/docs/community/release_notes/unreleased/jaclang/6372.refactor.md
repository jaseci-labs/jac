- **Refactor: Non-optional builder accessor in the native IR-gen pass**: The native code generator now keeps its active LLVM builder in a private optional field and exposes it through a non-optional `builder` accessor. Body-codegen helpers read the active builder without per-call None checks, and reaching one outside an active function body now raises a clear error instead of failing with an opaque NoneType dereference deep in IR emission. No behavior change for valid programs.

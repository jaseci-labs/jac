- **Fix: jac0 parses `static has` declarations (class-level constants)**: jac0's parser routed every `static` token to `_parse_funcdef` (which expected `static def`/`static can`), so `static has` hit a `ParseError` on the type-annotation colon and fell back to `jac2py`. The `jac2py` output retains `from jaclang...` imports that the embedded stdlib-only CPython in desktop hosts cannot load, which broke the loopback server in both `cef` and native `desktop` targets (`desktop_plugin.jac` was the affected file). The fix dispatches `static has` to `_parse_has` in both `_parse_item` and `_parse_enum_body`. (jaseci-labs/jaseci#6572)

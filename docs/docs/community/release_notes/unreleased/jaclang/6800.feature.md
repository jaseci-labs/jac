- **Feature: Client stubs carry import-path provenance for cross-module routing**: When a `.cl.jac` client calls a server function or spawns a walker that lives in a *different* module (via `sv import` or a plain cross-module import), the generated stub now stamps the call with its source-module stem as an optional trailing argument - `__jacCallFunction("create_order", args, "orders_app")` / `__jacSpawn("CreateOrder", "", fields, "orders_app")`. The client runtime resolves that stem against an optional `serviceRoutes` map supplied in the page's `__jac_init__` data and, when an entry exists, routes the request to `<prefix>/function/<name>` (or `<prefix>/walker/<name>`) instead of the flat `/function/<name>`. The map is opaque data the runtime applies generically - core carries no microservice or prefix policy. Same-module calls emit no provenance argument and, when no map is present, every call falls back to the flat path, so monolith output is byte-for-byte unchanged.

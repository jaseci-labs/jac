- **Fix: `by postinit` fields no longer raise a spurious `W1051` warning**: A field declared with `has x: T by postinit` reported a false `W1051` ("Expression type could not be resolved") on the `postinit` name, even though the field, the inline `def postinit`, and any separate `impl` all resolved correctly and go-to-definition worked. The type checker evaluated the `postinit` name as a standalone expression before the `HasVar` handler bound it to `__post_init__`. The checker now skips that trailing name and lets the handler bind it, the same way other context-only names are handled.

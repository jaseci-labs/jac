- **f-strings no longer drop quote characters from their literal text**: quotes written around an interpolation were silently deleted, so `f"two '{a}' and '{b}'"` printed `two 'x and y'` instead of `two 'x' and 'y'`, and `f'dq "{a}" and "{b}"'` lost its double quotes the same way. There was no error or warning, so any message built by interpolating quoted values came out subtly wrong. Quotes in an f-string's literal text are now treated as content rather than as string delimiters and are always preserved. Quotes inside `{...}` and raw f-strings (`rf"..."`) behave as before.

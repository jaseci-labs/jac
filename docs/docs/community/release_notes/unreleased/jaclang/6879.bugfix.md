- **Fix: narrowing into comprehension bodies inside `if x is not None` (#6731)**: A comprehension in the then-body of a guard such as `if b is not None { x: list[Box] = [b for _ in range(3)]; }` no longer raised a spurious `E1001 "Cannot assign list[NoneType] to list[Box]"`. The comprehension is wired into the CFG as a synthetic predecessor of the `if`, with a successor (`InnerCompr`) nested deep inside the then-body rather than at `body[0]`; `_classify_edge` matched the successor against `body[0]`/the else entry by identity, missed, and fell through to a "no-else ⇒ false" heuristic, applying the false-branch (`b is None`) narrowing. Edge classification now walks the successor's lexical parents to determine which branch subtree contains it — then-body ⇒ true edge, else-body ⇒ false edge — while real post-`if` siblings still use the unchanged fall-through heuristic. The true-positive direction is preserved: a comprehension inside `if r is None { ... }` that dereferences `r` still errors.

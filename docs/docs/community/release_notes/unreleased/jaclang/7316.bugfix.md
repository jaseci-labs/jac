- **Fix: TypeVar scope in detached `impl` blocks**: A type parameter declared on a generic archetype (`obj Box[T]`) is now in scope inside a separate `impl Box.method` block, in both the signature and the body. The type checker previously resolved type-variable scope purely lexically and reported a false `E1065: Type variable "T" has no meaning in this context`, even though the code ran correctly. Genuinely unbound type variables still report `E1065`.

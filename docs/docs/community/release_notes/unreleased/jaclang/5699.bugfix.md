- **Fix: Read-only walker stale-snapshot clobber in local mode (#5696)**: `SqliteMemory.sync` now uses the same hash-based dirty check as `MongoBackend.sync` and skips writeback for anchors the walker never mutated. Previously a long-running read-only walker would commit its stale snapshot at exit, silently overwriting writes from concurrent walkers, including the issue's deterministic reproducer where a read-only walker sleeping 1s would clobber another walker's write that landed during the sleep. `anchor.hash` is also refreshed after a successful merge-write to mirror the MongoDB path. Brings local-mode commit semantics in line with MongoDB-backed semantics so dev-vs-prod behavior matches.

- **Official jaclang Docker images**: every release now publishes `ghcr.io/jaseci-labs/jaclang:<version>` and `:latest`, and every main push refreshes the rolling `:dev` tag - multi-arch (amd64/arm64), built from the same binaries as the release assets, with jac's runtime pre-extracted so containers skip the one-time setup for any user id. `docker run --rm -v "$PWD":/app ghcr.io/jaseci-labs/jaclang:latest run main.jac` just works. CI/CD setup: no repository secrets or variables are needed - the `docker-image` job in `build-binaries.yml` authenticates to GHCR with the built-in `GITHUB_TOKEN` (job-scoped `packages: write`); the only manual step is one-time, after the first publish: set the `jaclang` package's visibility to **public** in the org's GHCR package settings (new packages are created private).

- **Lambda: parenthesized params, braced bodies, single-expression implicit return**: Lambda parameters are now always parenthesized and the body is always braced: `lambda (x: int) { x + 1; }`. The old colon-body form (`lambda x: expr`) and the paren-less form (`lambda x { ... }`) are removed. A body that is a single expression statement is the implicit return (`lambda (x: int) { x + 1; }` returns `x + 1`); multi-statement bodies use an explicit `return` (with none, they return `None`). Because the body is always braced, a `:` after a parameter is unambiguously a type annotation, so subscripted generics work in lambda param annotations (`lambda (d: list[int]) { d[0]; }`, closing #6773). A callable returning any type is now assignable where a `None`-returning (void) callable is expected, so single-expression event handlers like `onClick={lambda (e) { handle(e); }}` type-check.

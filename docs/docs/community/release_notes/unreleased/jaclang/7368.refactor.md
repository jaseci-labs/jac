- **Perf: Gate the native LLVM backend for non-native compiles**: `NaIRGenPass` and `NativeCompilePass` are only scheduled when a module actually needs native codegen (a `.na` file, auto-promote, or any native-context statement -- inline `na {}` blocks and native-prefixed decls). A plain Python/client compile served by `jac start` no longer maps the ~130 MB `libjacllvm.so` ctypes binding, saving ~84 MB RSS + ~130 MB VSZ per non-native process. Mixed sv/na modules still JIT into the native engine so live sv->na calls resolve. The scale deploy/observability factories are also imported lazily inside the `deploy_*` commands, so a local `jac start`/`jac serve` dev server no longer compiles those subtrees.

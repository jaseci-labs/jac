- **Fix: Relative imports now work in a project's entry file**: An entry file that reached a sibling module with a relative import (for example `import from .services.greeter { greet }`) failed at runtime with `attempted relative import with no known parent package`, even though `jac check` reported the file as valid. The entry file is now given the package context it needs, so relative imports resolve the same way whether the file is the one being run or is imported by something else.

- **Fix: exponential slowdown in comment injection pass on deeply nested DocIR DAG**: the DocIR is a DAG where `IfBreak` break/flat branches share child objects, so the pure structural helpers `_remove_redundant_lines`, `_get_tokens`, and `_doc_line_span` re-traversed shared subgraphs up to 2^k times on nested `IfBreak` nodes. These three (being pure functions of their node) are now memoized with object-identity caches cleared per transform, so each node is measured at most once per pass. The comment-placement walk `_process` is deliberately left un-memoized: it consumes comments from a shared store and depends on its incoming context, so caching it on node identity would misattribute or duplicate comments across the DAG. On one 2.3k-line file the comment-injection pass drops from ~6.5s to ~2.7s (~2.4x) and end-to-end `jac fmt` on that file from ~8.5s to ~2.5s; a whole-repo `jac fmt` (1306 files, dominated by small files) is meaningfully faster overall. Output is byte-for-byte identical to the previous formatter; the memoization only affects speed.

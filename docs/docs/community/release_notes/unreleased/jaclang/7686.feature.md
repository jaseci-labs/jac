- **Native dynamic-object-model constructs get a dedicated by-design diagnostic (E5093) and literal-name reflection lowers (#7685)**: the two silent-miscompile holes are closed -- statement-position lowering failures (`setattr(p, "nope", 5);` compiled clean and did nothing) and `del p.x` attribute deletion (fell off the dispatch chain) now always diagnose; every dynamic object-model construct (reflection builtins with runtime names, `delattr`, `vars`, `globals`/`locals`, `__dict__`, `__class__` assignment -- now anchored on the construct instead of the RHS -- class-object monkey-patching, 3-arg `type()`, and member stores through unresolvable `any` receivers) reports the new `E5093` "not supported in native code by design" with help naming the static alternative, distinct from E5090 (not yet implemented) and E5092 (shape failed to lower); statically recognizable shapes report from the capability gate before codegen. Literal-name reflection on declared fields is recognized as static and lowers to plain field operations with Python-congruent semantics: `setattr(p, "x", v)` is `p.x = v`, `getattr(p, "x")` is `p.x`, `getattr(p, "nope", d)` yields `d`, and `hasattr(p, "x")` folds to a constant -- pinned by py/na cross-backend equivalence fixtures. Checker `E1030` additionally teaches the sealed-layout design when it fires in native code, and the native pathway reference documents the object-model contract.

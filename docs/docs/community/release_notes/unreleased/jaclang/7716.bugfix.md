- **Fix: domainless redeploys keep the ingress host binding**: The per-app ingress render resolved its domain only from config, so a redeploy driven without the domain in config (it may only ever have been supplied at `enable_tls` time) wiped the host binding and silently broke HTTPS. The render now falls back to the `jac-scale/domain` annotation `enable_tls` maintains, using the same precedence `enable_tls` itself uses, for both the main and redis-insight ingresses.

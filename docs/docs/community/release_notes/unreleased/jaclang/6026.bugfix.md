- **Fix: Module-level version-guarded typeshed symbols (e.g. `datetime.UTC`) now resolve correctly**: Symbols defined inside `if sys.version_info >= (...)` blocks at the module level of stdlib stubs were silently dropped from the symbol table, causing them to resolve as `Unknown` and producing false-positive errors such as `E1053: Cannot assign <Unknown> to parameter 'tz'` when using `datetime.UTC`; the existing `_flatten_version_conditionals` helper that already handled this inside class bodies is now also applied to module bodies.

- **Feat(jac2c): transitive import chains in cross-module C emission**: `jac jac2c` now walks the whole import graph when emitting a module's translation unit, not just its direct imports. A top module `A` that imports from `B`, where `B` imports a reference type from `C`, receives and drives a `C`-defined value that reaches `A` only through `B`; its emitted translation unit now forward-declares the leaf `struct` and `extern`s its constructor and methods, and lowers method calls on those values to the mangled cross-module symbols (`Cell__bump` / `Cell__get`) instead of leaving them unresolved. `CGenPass.classify_imports` was direct-imports-only while the graph-discovery (`discover_import_graph`) and shared facts/layout (`get_shared_type_facts` / `get_shared_layout_registry`) were already transitive; this closes that gap so a per-module unit's extern decls and type-ids match the whole-program picture. Proven by `test_jac2c_transitive.jac`: a three-module `chain_leaf` → `chain_mid` → `chain_main` chain whose linked binary output matches the stock interpreter.

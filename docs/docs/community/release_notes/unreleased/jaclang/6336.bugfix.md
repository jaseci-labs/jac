- **Type checker: fewer false positives on context managers and iterable unpacking**: `with console.status(...) as s` no longer raises a spurious "cannot be used in a `with` statement", and unpacking a non-tuple iterable — `(b0, b1) = some_bytes` — now types each target as the element type (`int` here) instead of leaving them untyped, so bitwise and arithmetic operations on them stop reporting bogus errors.

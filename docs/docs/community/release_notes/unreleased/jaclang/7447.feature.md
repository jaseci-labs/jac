- **Perf: Fast-path `jac --version`**: `--version` is now resolved before the full CLI is imported, dropping it from ~1.3s to ~0.5s. Project config, provider, and template initialization is deferred until after help/version exits, so ordinary commands also skip that work.
- **Perf: On-demand feature command loading**: Feature plugins (client/scale/shadcn/byllm) are now imported only when needed: all of them for bare/`--help`, just the matching one for `jac setup`/`scale`/`retheme`/`model`, and none for ordinary commands, saving ~260ms on every non-feature invocation. Routing is centralized in `register_feature_commands(raw_argv)` backed by a declarative `FEATURE_COMMAND_MODULES` map, so adding a feature is now a one-line data entry.
- **Perf: Manifest-backed CLI routing (#7447)**: Lightweight invocations (`jac`, `jac --help`, `jac --version`, unknown commands, tombstones) now render from a generated command manifest instead of building the full argparse registry or importing every command module. Known commands dispatch through a per-command lazy loader that imports only the selected handler and its feature bootstrap.
- **Perf: Lazy `jaclang` package init (#7447)**: `import jaclang` no longer eagerly loads the runtime and compiler (~350ms on warm cache). `JacRuntime`, `JacRuntimeInterface`, and `compiler` remain available via PEP 562 lazy exports, and `cli_boot` routes informational paths through manifest helpers that avoid importing `console.jac` (which would pull in the full runtime via the meta importer). `jac --version` and `jac --help` drop from ~945ms / ~1157ms to ~360ms / ~380ms on a warm dev tree.

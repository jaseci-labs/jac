- **Compiler: FilterSpec storage pushdown for `[?:Type, field op const]` filters**: The compiler now emits a `FilterSpec` wrapper (type name, MongoDB-style predicate dict, and lambda fallback) instead of a plain lambda when a `[?:Type, field op const]` filter has extractable field predicates. Type-only filters (`[?:User]`) keep the plain lambda so the `TopologyIndex` fast-path is unaffected. The runtime's `refs()` detects a `FilterSpec`, constructs a `QueryPlan` (from `jaclang.runtimelib.query_plan`), and routes it through `Memory.execute_plan()` when the backend declares `field_pushdown` or `type_pushdown` capability - so `node --> [?:User, age < 28]` becomes a single server-side query (MongoDB `collection.find()`, Postgres jsonb GIN, etc.) rather than a full graph scan. The `id_in` candidate set and `slice` are also pushed when the backend declares the corresponding capabilities.

- **Compiler: FilterSpec storage pushdown for `[?:Type, field op const]` filters**: The compiler now emits a `FilterSpec` wrapper (type name, MongoDB-style predicate dict, and lambda fallback) instead of a plain lambda when a `[?:Type, field op const]` filter has extractable field predicates. Type-only filters (`[?:User]`) keep the plain lambda so the `TopologyIndex` fast-path is unaffected. The runtime's `refs()` detects a `FilterSpec` and, when an L3 backend with `query_by_type` is present (e.g. jac-scale's `MongoBackend`), delegates the query server-side before any Python-side iteration; `node --> [?:User, age < 28]` becomes a MongoDB filter rather than a full graph scan.

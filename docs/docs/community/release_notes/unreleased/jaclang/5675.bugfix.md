- **Parser: Actionable Diagnostics for Malformed Node Connection Operators**: The parser now emits five new targeted error codes (E0025-E0029) when a node connection operator is used incorrectly, replacing generic parse errors with messages that name the exact problem and suggest the correct form. For example, writing `n1 ++> Friend n2` (attempting to attach an edge type to a simple operator) now produces `E0027: '++>' does not take an edge type - use '+>: <EdgeType> :+> <node>'`, and writing `n1 <+: Friend +> n2` (missing colon on the closing token) produces `E0026: Expected ':<+' or ':+>'` pointing directly at the offending token. Error recovery via `synchronize()` ensures only the root-cause diagnostic is shown, suppressing cascade noise.

- **Feature: bundled native `base64` (#6978 Phase 3)**: `import from base64 { ... }` now resolves on the native (`nacompile`/JIT) pathway to a bundled pure-Jac (Mechanism B) module (`na_stdlib/base64.na.jac`) with no FFI floor and no dependencies, so it links on every native target. The RFC 4648 codecs `b16encode`/`b16decode`, `b32encode`/`b32decode`, `b64encode`/`b64decode` (with `altchars`), `standard_b64encode`/`standard_b64decode`, `urlsafe_b64encode`/`urlsafe_b64decode`, and the RFC 1924 `b85encode`/`b85decode` (the alphabet CPython's `base64.b85encode` uses) produce byte-identical output to CPython and round-trip its output. The codecs are a big-endian bit-accumulator over `bytes` primitives that grows the result in a `list[int]` and converts once with `bytes(...)` (length-based, NUL-safe), so there is no big-int dependency. `b16decode`/`b32decode` accept the `casefold` flag. SCOPE: decoders validate the alphabet and decode well-formed input congruently; strict length/padding validation (CPython's `validate=`/`binascii.Error`) and the Ascii85 (`a85encode`/`a85decode`) variant are follow-ups. Pinned sv<->na congruent by the new `prim_base64` equivalence fixture (digest + length of each encoding plus a decode round-trip across every padding boundary).

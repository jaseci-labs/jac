- **Fix: Detect JS ternary operator (`? :`) with E0029**: The parser now emits diagnostic E0029 when a JavaScript-style ternary expression (`condition ? then : else`) is used in expression position, replacing a cascade of 30+ confusing downstream errors with a single precise diagnostic pointing directly at the `?`. Users are guided to use Jac's if-expression form (`value_if_true if condition else value_if_false`) instead. Legitimate null-safe access (`?.` and `?[]`) is unaffected.

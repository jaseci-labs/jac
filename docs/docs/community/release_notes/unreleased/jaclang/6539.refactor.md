- **Refactor: Extract FrameworkBackend seam from EsastGenPass**: Client codegen (cl.jac / cl {} → JavaScript) is no longer hard-wired to React. A new pluggable `FrameworkBackend` interface and framework-neutral reactive-intent vocabulary (`StateField`, `RefField`, `StateUpdate`, `Effect`, `AsyncBoundary`) allow per-framework lowering. All React-specific emission (`useState`, `useRef`, `useEffect`, `setCount`, `<JacAwaiting>`, `<JacClientErrorBoundary>`) moves into `ReactBackend`. A `[client] framework` config section (default `"react"`) selects the backend at compile time. `E5081` is raised for unknown framework names.

- **interopbench: marshalled-boundary benchmark suite (`jac/examples/interopbench`)**: two experiment families mirroring the ownbench layout: family 1 (`iop_*`, `harness/measure.jac`) covers mixed-JIT `sv↔na` scalar calls, the symmetric `iop_symmetric` caller/callee pair, zero-copy `NativeListView` consumption, and optional C-ABI fixtures; family 2 (`xop_*`, `harness/xbench.jac`) covers loopback RPC, generated Node clients, and wasm hosts. Shared `harness/common.jac` owns the strict output parser, aggregation, oracle comparison, and schema-versioned JSON. Per-family entry points (`run_bridges.sh`, `run_xruntime.sh`, `ci_bridges.sh`, `ci_xruntime.sh`, umbrella `run_all.sh`) plus `test_interopbench_bridges.jac` wire the dependency-light cells into the compiler regression net; default measurements use 30 invocations and 5000/500 work/call sizes. Results are schema version 2: every document carries machine/OS/python provenance and each cell records `samples` and a robust `iqr_ns` spread; invocations are interleaved per variant (paired sampling) and cross-runtime cells that do not share a clock label `timing_source` per variant. Measurement units and labels are corrected to match what each cell actually measures - the `iop_ffi_*` cells report per-foreign-call time (`m:per_ffi_call_ns`, dividing by the true foreign-call count), the C fixture carries `_Static_assert`ed struct sizes, and the docs now state plainly that `iop_call`/`iop_symmetric` are execution-placement (not isolated-boundary) benchmarks, `iop_cb` is the clean boundary cell, `xop_feed` is a scalar RPC floor rather than a typed feed, and `xop_wasm_call` compares asymmetric host loops.

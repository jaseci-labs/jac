- **`jac ai`: Persistent sessions (resume / continue / named)**: Conversations now survive process restart. `jac ai -c` resumes this project's most recent session, `jac ai --resume <id>` resumes a specific one, and `jac ai --session <name>` opens or creates a named session. The agent's cross-turn memory (ledger), display toggles, working directory, and accumulated token totals are restored on resume, and the session is auto-saved after every turn. Sessions are stored as JSON under the centralized jac cache root (`~/.cache/jac/ai/sessions/` on Linux, alongside the JIR cache) with a `JAC_AI_CACHE` override, and scoped to the project directory; writes are atomic and corrupt files are skipped rather than crashing the agent. `--resume`, `--continue`, and `--session` are mutually exclusive.
- **`jac ai`: Session transcript scrollback and management commands**: On resume, prior turns are printed as a readable scrollback (objective + each turn's final phase answer) so you can pick up where you left off; this transcript is display-only and is never fed back to the model, which still sees only the ledger. New in-REPL commands manage sessions: `/sessions` lists this project's sessions (current one marked), `/resume <id|name>` switches to another session in place, `/name <name>` names or renames the current session (names are unique within a project), `/cost` shows cumulative token usage and tool counts, and `/forget` deletes the current session file after confirmation and starts a fresh one. `/clear` now also wipes the persisted transcript and totals so a later resume of the same session stays cleared (the session id and name are kept).

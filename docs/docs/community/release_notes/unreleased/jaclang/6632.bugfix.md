- **Type checker: five false-positive classes eliminated**: (1) an assignment with an explicit annotation (`x: T = ...`, `has x: T = ...`) now absorbs an `Any` source - the annotation is the developer typing the boundary, so Python interop like `row: tuple[str, str] | None = cursor.fetchone()` checks cleanly while silent `Any` flows (call arguments, returns, unannotated reassignments) stay strictly rejected; (2) class objects are accepted where a `Callable` is expected via their constructor signature (`json.dumps(data, default=str)`); (3) recursive type aliases with quoted forward refs (`JsonValue = ... | list['JsonValue']`) resolve correctly instead of degrading to `Literal["JsonValue"]`; (4) unsolved TypeVars no longer leak out of generic calls (`list(x)` with an `Any` argument used to surface `_T` in downstream diagnostics); (5) `any?.attr` stays `Any` instead of collapsing to `NoneType`, `cls.init` resolves to `__init__` on Python-declared classes, and the qualified `typing_extensions.TypeAlias` stub form is recognized.

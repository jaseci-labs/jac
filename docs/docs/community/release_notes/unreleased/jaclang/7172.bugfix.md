- **Fix: JIR round-trip preserves `FuncCall.callee_decl`**: `callee_decl`, the checker-stamped pointer from a function/method call to its resolved `Ability`, is now a serialized JIR field. Previously it was dropped on JIR reload (it was declared `by postinit` and absent from the serialization schema), which reset it to `None` for warm (JIR-cached) modules. The client (`cl`/JS) backend reads `callee_decl` to lower keyword arguments to ordered positional args, so a warm-loaded client dependency lowered kwarg method calls as a packed object (`x.m({b: 1, a: 10})`) instead of positional (`x.m(10, 1)`), diverging from the cold build. Serializing the pointer eliminates the warm≠cold divergence at the source, so the dependency re-inference sweep that #6665/#6666 added to re-stamp it is no longer required for correctness. (#6666)

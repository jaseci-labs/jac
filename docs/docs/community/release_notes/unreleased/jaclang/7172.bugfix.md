- **Fix: JIR round-trip preserves `FuncCall.callee_decl`**: `callee_decl`, the checker-stamped pointer from a function/method call to its resolved `Ability`, is now a serialized JIR field. Previously it was dropped on JIR reload (it was declared `by postinit` and absent from the serialization schema), which reset it to `None` for warm (JIR-cached) modules. The client (`cl`/JS) backend reads `callee_decl` to lower keyword arguments to ordered positional args, so a warm-loaded client dependency lowered kwarg method calls as a packed object (`x.m({b: 1, a: 10})`) instead of positional (`x.m(10, 1)`), diverging from the cold build. Serializing the pointer eliminates the warm≠cold divergence for **intra-module** calls, whose callee lives in the same JIR blob and round-trips as a real node ref. **Cross-module** calls still come back as `None` (the callee lives in a different blob and encodes as an external ref), so the codegen-entry re-inference sweep (`ensure_callee_decl_restamped`) is retained at the ES/native codegen and `client_bundle` sites to re-stamp those. Relatedly, `is_module_cache_valid` now validates cached JIR purely by content hash and drops the mtime fast-path, since a stale-but-newer-mtime JIR would reintroduce the same divergence. (#6666)

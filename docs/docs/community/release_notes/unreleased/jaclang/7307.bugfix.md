- **Fix: jac-scale Redis pod resources are configurable and no longer OOMKilled by default**: The provisioned Redis pod's container CPU/memory requests and limits were hardcoded (limit `cpu 500m / memory 256Mi`), too small for real RSS once BGSAVE fork copy-on-write, fragmentation, and buffers pushed it past the cgroup limit, causing recurring OOMKills and client `Connection refused` errors. You can now size the pod from `jac.toml` `[scale.database]` via `redis_cpu_request`, `redis_cpu_limit`, `redis_memory_request`, and `redis_memory_limit`. Defaults are raised to `250m/1000m` CPU and `512Mi/1Gi` memory to give headroom above the default `redis_max_memory` (200mb), and a warning is logged at deploy time when `redis_memory_limit` leaves less than roughly 2x that headroom.

- **Fix: redeploy no longer silently breaks HTTPS on TLS-enabled apps**: On the per-app ingress path, a redeploy re-rendered the ingress with an empty rule host, wiping the host binding `enable_tls` had created — nginx then served the default fake certificate on 443 while every status signal stayed green. The deploy render now reads the live ingress first and, when it carries the TLS state `enable_tls` wrote (`spec.tls` / ssl-redirect), re-emits the rule host, `spec.tls` block and TLS annotations for both the main and redis-insight ingresses, from the same helpers `enable_tls` uses. Apps that never ran `enable_tls` render exactly as before (the pre-TLS catch-all rule is preserved), and already-broken apps self-heal on their first redeploy after this fix.

- **jac-client: desktop `--dev` HMR (native webview)**: `jac start --client desktop --dev` builds the native host once, serves the `cl` UI from Vite on 127.0.0.1, and recompiles on every `.jac` save. Walker/function calls still route through the embedded in-process runtime, so RPC matches the packaged build. The Vite dev origin is wired to the host purely via env vars (`JAC_DESKTOP_DEV_PORT` / `JAC_DESKTOP_BROKER_PORT`) read at boot, so the prod build path is unchanged. CEF `--dev` raises explicitly instead of faking it (CEF routes RPC over the loopback broker). `--api-port` now pins the broker port (the native target's in-process RPC surface), matching mobile's precedence. The shared per-`.jac` recompile handler (`_ClientHmrHandler`) and the Vite-dev compilation prep (`prepare_vite_dev_compilation`) are now single helpers shared across the desktop and mobile `--dev` targets, and `--dev` no longer runs a redundant production web bundle (the webview loads from Vite).

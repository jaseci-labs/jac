The fleet rollout watcher now reschedules a wedged pod once before failing:
a pod whose image pull keeps failing, or that has sat five minutes in
ContainerCreating with FailedMount events, is deleted so its ReplicaSet can
place a replacement (usually off the bad node). One reschedule per service,
three per rollout; if the replacement wedges too, the rollout fails exactly
as before, and merely slow pods are never touched.

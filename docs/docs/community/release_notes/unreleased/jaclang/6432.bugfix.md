- **Fix: native `dict.pop` and `complex` codegen**: `dict.pop` no longer emits a malformed LLVM PHI node on the key-missing path that raises `KeyError` (the runtime-raise splits the block, so the real predecessor is now recorded). `complex` now works end to end: the `complex` annotation lowers to a boxed `JacComplex*` instead of falling through to `i64`, so `.real`/`.imag` and `+`/`-`/`*` arithmetic return correct values instead of `0.0`. (jaseci-labs/jaseci#6432)
- **Fix: native match-capture use-after-scope crash**: pointer-typed `match` captures (a `*`-star capture like `case [0, *rest]`, or a plain `case x` binding a list/str/object) are only assigned on the matching path, but their stack slot was left uninitialized when that case wasn't taken. Function-exit refcount cleanup then released the garbage pointer, intermittently segfaulting. The capture slots are now null-initialized in the entry block, so cleanup on a non-taken case is a safe no-op. (jaseci-labs/jaseci#6432)

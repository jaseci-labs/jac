Added type-aware `.jab` artifact core (Issue #6335 PR 1):

- **`jab.kind` enum**: Every `.jab` now carries an explicit `kind` in its manifest (`script`, `service`, `py-package`, and 8 forward-declared kinds). The toolchain reads the kind at consume time to decide how to handle the bundle (run / serve / attach).
- **Capability flags**: A bitfield (`CAP_HAS_SERVER`, `CAP_HAS_CLIENT`, `CAP_HAS_ENTRY`, `CAP_ATTACHABLE`, etc.) describes what the bundle contains, enabling fast dispatch without parsing the full payload.
- **`[project] kind`** in `jac.toml`: Explicit project kind declaration; auto-detected from source when omitted.
- **`read_program_jir()`**: Tier-0 `.jab` deserializer for loading pre-built bundles.
- **`load_program_bundle()`**: Compiler method to load and validate `.jab` files (Python version check, per-module bytecode extraction).
- **`jac run app.jab`**: Running a `.jab` directly is now supported -- loads bytecodes from the bundle, validates kind (rejects packages with no entrypoint), and executes.

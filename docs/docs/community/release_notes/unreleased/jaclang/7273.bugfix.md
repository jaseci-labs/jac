- **Fix: `[plugins.client]` TOML sections silently dropped on config reload**: `_parse_toml_data`, `merge_from_toml_file`, and `apply_profile` now scan `data["plugins"]` for client/byllm/scale/mcp/desktop entries in addition to the top-level `[client]` etc., so a `[plugins.client]` section written by a save round-trip is no longer lost. Client npm dependency injection (`@tanstack/react-form`, `zod`, …) is now gated on the resolved project kind (`web-app`, `web-static`, `desktop`, `mobile`) rather than the presence of a `[client]` section, so a client project builds even when a custom jacpack template omits that section.

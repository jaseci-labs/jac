- **f-strings now decode escape sequences next to a double-quote**: an escape stopped being decoded whenever the surrounding literal text also contained a `"`, so `f'x "{n}" \x41{v}'` printed `x "A" \x411` instead of `x "A" A1`. Octal escapes such as `\101` were affected the same way, while `\n`, `\t`, `\r` and `\\` kept working, and moving the `"` elsewhere made the same escape decode correctly. Escape sequences in f-string text are now decoded consistently regardless of any quotes beside them.

- **Refactor(jac2c): back TypeFacts with the real TypeEvaluator**: On the emit path, `TypeFacts.type_of` now consults the checker's `TypeEvaluator` first (via each expression node's cached type, else a lazy `get_type_of_expression`) and translates its structured verdict into the facts layer's vocabulary: primitives come straight from the checker (the scalar lattice it owns authoritatively, where the old `expr_type_name` copy had drifted), and a user archetype's `ClassType` is mapped through the shared `TypeIdIndex` to its module-qualified tid (the checker's bare `class_name` cannot distinguish two modules' same-named types, but its `ClassType` points at the correct defining archetype, so cross-module identity from #7145 is preserved). The syntactic re-inference remains only as a fallback for what a single-TU evaluation leaves unresolved (e.g. imported functions typed as `Unknown`). Consumers that build facts without a program handle (the ownership pass) keep the pure syntactic behavior. This removes the parallel-type-system drift the facts layer risked without changing emitted C (verified by the emit differential, name-collision, and lift-oracle suites).

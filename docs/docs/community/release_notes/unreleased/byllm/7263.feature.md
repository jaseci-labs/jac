- **Terminal tool designation (`finish_tool=`)**: `by llm(tools=[...], finish_tool=fn)` designates one of your own functions as the terminal tool that ends the ReAct loop and yields the call's output. byLLM enforces it (re-prompts once if the model tries to end the loop without calling it) and skips injecting its generic `finish_tool`, giving `-> str` agents a guaranteed completion step instead of a silent early exit. Non-blocking warnings flag a reserved `finish_tool` name in `tools=[...]` or an undesignated `finish`-named tool.
- **Optional tool parameters are omittable**: a tool parameter with a default (for example `finish(answer: str = "")`) is no longer marked `required` in the generated tool schema, so the model may omit it instead of padding a value.

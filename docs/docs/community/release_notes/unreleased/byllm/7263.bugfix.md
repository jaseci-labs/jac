- **Fix: structured output no longer silently degrades to a string**: when the ReAct loop hit `max_react_iterations`, a call declared to return a structured type could hand back a raw string with no error. The forced final answer is now validated against the return type and raises into the typed-output retry loop instead.
- **Fix: finish-tool output validation feeds the retry loop**: a finish-tool output that failed type validation raised a hard `FinishToolError` that bypassed the correction retry. It is now caught in the typed-output retry loops and carries `raw_output`, so the model is re-prompted with the rejected value.
- **Fix: `str`-return agents can enforce a required terminal tool**: recovery that re-prompts for the terminal tool was gated off for `str` return types, so a designated `finish_tool` was never enforced for the common `-> str` agent. It now fires whenever a terminal tool is designated.
- **Fix: mock tool calls carry runtime tool flags**: `MockToolCall` wrapped a fresh, disconnected `Tool`, unlike real dispatch which resolves calls through `mt_run.get_tool()`. The mock now re-points each call to the registered tool, so it mirrors real behavior under test.

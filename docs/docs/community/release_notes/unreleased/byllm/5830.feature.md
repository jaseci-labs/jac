- **Add: Built-in local LLM runtime via `local:<alias>`**: Run Gemma 4 E4B/E2B and Qwen 3.5 4B in-process through `llama.cpp` with no API key, no daemon, and no proxy server. Weights download lazily into `~/.cache/jac/models/<alias>/` on first use and are managed via the new `jac model list/pull/rm` command. Configurable under `[plugins.byllm.local]` (`default_alias`, `n_ctx`, `n_gpu_layers`, `n_threads`, `verbose`, `auto_download`). `llama-cpp-python` and `huggingface_hub` are now base dependencies of byLLM so the zero-config local fallback works out of the box; the `[local]` extra is preserved as an empty alias for backwards compatibility.

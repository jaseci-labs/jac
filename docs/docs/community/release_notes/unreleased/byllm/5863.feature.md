- **feat(byllm): pass-through arbitrary LiteLLM call params from byLLM**: `make_model_params` now merges any unreserved key from `call_params` (across all surfaces: `jac.toml`, `Model(call_params={...})`, `model(...)`, and `by llm(...)`) directly into the `litellm.completion` kwargs. byLLM-internal keys (`compaction_enabled`, `max_react_iterations`, `on_iteration`, `parallelize`, `logging`, `ctx_window`, etc.) are excluded via a reserved-key set. Explicit curated keys (`temperature`, `max_tokens`, `tool_choice`) keep their existing defaults and precedence. This unblocks provider-specific features like `top_p`, `seed`, `reasoning_effort`, `thinking`, `extra_headers`, `extra_body`, `timeout`, and `cache`.

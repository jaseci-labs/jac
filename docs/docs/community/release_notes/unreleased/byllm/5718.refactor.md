- **Change: parallel-tool scheduling hints are now opt-in.** The `[PARALLEL-SAFE]`/`[SEQUENTIAL]` tag injection on tool descriptions and the scheduling-rules block in the system prompt now only fire when `parallel_hint=True` is passed to `by llm()`. Previous default was `True` (opt-out via `parallel_hint=False`); modern LLMs already batch independent tool calls, and the hint costs prompt tokens on every request. Runtime behavior (`mark_serialize`, `dispatch_batch` parallel/sequential gating) is unchanged. Addresses item 2 of #5711.

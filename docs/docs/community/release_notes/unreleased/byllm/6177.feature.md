- **Add: `system_prompt` parameter on `by llm()` for per-call SYSTEM message control**: New optional `system_prompt` kwarg on `by llm()` accepts a `str` or zero-arg `Callable[[], str]`. The callable form is invoked each turn, enabling dynamic per-turn system prompts (mode-aware, session-aware) without project-wide config. Extends — does not replace — the base SYSTEM content; the final SYSTEM is `[jac.toml [plugins.byllm].system_prompt or built-in SYSTEM_PERSONA] + per-call system_prompt + INSTRUCTION_TOOL (when tools are present)`. Precedence: per-call > `jac.toml` > default. Resolves the duplication problem in conversational byllm apps (`by llm(conversation=<list>)`) where the function's sem was folded into every persisted user message via `Tool.get_func_description(caller)` — routing the standing persona through `system_prompt` keeps it in the cacheable SYSTEM slot (excluded from persistence via `_scaffolding_ids`) and out of chat history, so per-turn payload no longer grows linearly with conversation length. Fully backward compatible — the parameter is optional and existing `by llm(...)` calls are unaffected.

- **Fix: serve `/static/assets/*` from the project tree on the microservice gateway**: the gateway now resolves `/static/assets/<path>` from the project's source `assets/` directory when the asset is absent from the client build output, mirroring the monolith server. Assets referenced by a literal URL (e.g. a logo under `assets/`) now load behind the microservice gateway without per-project build config or static mounts. Nested sub-paths are supported and the lookup is jailed to the `assets/` directory (#6872).

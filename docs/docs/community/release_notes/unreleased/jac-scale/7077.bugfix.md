- **Fix: microservice serve no longer crashes on a missing `requests`**: The observability `prometheus_client` module imported `requests` at module level, pulling it into the serve import chain even though it is outside the base serve closure. In pods without `requests`, `server.serve` failed to import, `get_api_server_class` silently swallowed the error and fell back to the core server, and the fallback then died with a misleading `JFastApiServer ... does not support the context manager protocol`. `requests` is now imported lazily inside the Prometheus query functions (and a genuinely missing `requests` degrades to `None`), and `get_api_server_class` logs which module was missing instead of swallowing the error.

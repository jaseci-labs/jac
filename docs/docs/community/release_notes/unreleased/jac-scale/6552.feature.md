- **Feature: Ops Console - Deploy Health + read-only K8s RBAC foundation (V-02)**: A new **Operations -> Deploy Health** admin page reads the Kubernetes API (namespace-scoped, read-only) to show, kubeconfig-free, every deployment's ready/desired replicas + rollout status (complete / progressing / stuck) and every pod's phase, ready containers, restart count, node, and per-container spec-image-vs-resolved-imageID. This lands the shared Ops Console foundation: a gateway-runtime in-cluster client helper (`admin/k8s_ops.jac`, degrades to a clean "unavailable" off-cluster, never a 500) and an opt-in namespace-scoped read-only `Role` + `RoleBinding` bound to the gateway ServiceAccount, provisioned by the microservice target when `[plugins.scale.kubernetes].ops_console = true` (no ClusterRole, so per-app-namespace multi-tenancy stays intact). Pure reshape helpers are unit-tested against mocked API payloads.

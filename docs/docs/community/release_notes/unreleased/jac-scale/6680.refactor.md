- **Refactor: `Db` type-switch god-object replaced with per-backend subclasses (#6680)**: The single `Db` obj (30 `self.db_type==` branches, 18 `NotImplementedError` stubs) is split into a slim `Db` base that declares only the four universal `get`/`set`/`delete`/`exists` methods, with `MongoDb(Db)` and `RedisDb(Db)` holding their backend-only methods, mirroring the `Storage` to `S3Storage` precedent. `kvstore()` now returns the concrete subtype (`MongoDb | RedisDb`), so cross-backend method calls become compile-time type errors instead of runtime `NotImplementedError`s, and the duplicated read-only-collection validation is hoisted into shared `_db_helpers.jac`. A `FirestoreDb(Db)` stub lands as the extension point for the upcoming Firebase PR.

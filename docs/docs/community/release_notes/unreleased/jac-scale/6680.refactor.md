- **Refactor: `Db` runtime type-switch dispatch replaced with polymorphic subclasses (#6680)**: The `Db` obj's `self.db_type == ...` dispatch (30 branches) is replaced with per-backend subclasses `MongoDb(Db)` and `RedisDb(Db)`, mirroring the `Storage` to `S3Storage` precedent. **This is an internal refactor with no public API change**: `Db` still declares the full method surface and the `client`/`db_name`/`db_type` fields, `kvstore()` still returns `Db`, and calling a backend-specific method on the wrong backend still raises `NotImplementedError` at runtime (now via inherited base defaults instead of an explicit branch). Internally, per-backend method declarations and bodies move into `mongo_db.jac`/`redis_db.jac` and their impl files, the duplicated read-only-collection guard is hoisted into shared `_db_helpers.jac`, and ObjectId parsing is deduped via `_as_object_id()`. The `find_by_id`/`update_by_id`/`delete_by_id` methods now route through `_as_object_id()` too, so a non-ObjectId `id` reports not-found (`None`/zero-count result) instead of raising `bson.errors.InvalidId`, matching the existing `get`/`delete`/`exists` behaviour. A `FirestoreDb(Db)` stub lands as the extension point for the upcoming Firebase PR.

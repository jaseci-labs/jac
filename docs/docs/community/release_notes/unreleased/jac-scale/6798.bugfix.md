- **Fix: `--no-image` pods install the scale plugin's runtime deps + get a startup probe**: the in-pod bootstrap installed bare `jac-scale`, so the `jac-scale:scale` plugin failed to load in every pod (gateway crashed on `jac scale gateway`, service pods 404'd on `/healthz/*`). It now installs `jac-scale[all]` plus `requests` (a scale-plugin runtime dep jac-scale ships only in its `test` extra) and `jac-client` when present locally. No-image pods also get a startup probe so the multi-minute in-pod install/compile isn't SIGKILLed by the liveness probe before the app first serves.

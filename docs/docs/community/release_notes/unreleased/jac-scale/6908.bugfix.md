- **Fix: `KEDAAutoscaler.destroy()` now deletes owned `TriggerAuthentication` resources before removing the `ScaledObject`**: Previously, `destroy()` only deleted the `ScaledObject`, leaving any `TriggerAuthentication` resources created by `apply()` as permanent orphans in the namespace with stale secret references. `destroy()` now reads the live `ScaledObject` to discover referenced `TriggerAuthentication` names from `spec.triggers[*].authenticationRef.name`, deletes each one (404-safe), then deletes the `ScaledObject`. If the `ScaledObject` is already absent, `destroy()` returns early without calling delete at all.

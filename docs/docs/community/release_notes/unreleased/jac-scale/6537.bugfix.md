- **Fix: node + edge creation is now crash-atomic in the Mongo backend (#6488)**: `MongoBackend.sync()` flushed a new node's writes (source edge list, edge document, target node document) inline in working-set order, which always wrote the target node last. A process killed mid-sync (rolling deploy, OOM, pod eviction → SIGKILL) after the edge but before the target node left a dangling edge pointing at a node that was never persisted, breaking every traversal that crossed it (`NodeAnchor [...] is not a valid reference!`). `sync()` now flushes in dependency order — nodes, then edges, then the edge-list/field updates that reference them — so a mid-sync crash can at worst leave a harmless orphan node, never a dangling edge. (jaseci-labs/jaseci#6488)

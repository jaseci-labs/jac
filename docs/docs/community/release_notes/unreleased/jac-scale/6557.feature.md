- **Feature: Ops Console - Controlled Actions (V-10, Tier 2)**: A new **Operations -> Actions** admin page exposes four mutating operations behind a typed confirmation (re-type the exact target name to arm): rollout-restart and scale a deployment, delete a pod, and unstick a Terminating PVC (finalizer clear). Every call is admin-gated, audit-logged server-side (actor + action + target + namespace + result), and needs the separate mutation RBAC the microservice target only grants when `[plugins.scale.kubernetes].ops_console_mutations = true` (without it the K8s API returns 403 and the action reports it, never a 500). Redeploy and ingress-backend flips are intentionally excluded - redeploy goes through the normal build/apply pipeline (the Role has no create verbs). Pure patch-body builders + input validators (replica clamp, non-empty name) unit-tested.

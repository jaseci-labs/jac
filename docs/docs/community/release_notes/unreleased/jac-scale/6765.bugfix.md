- **Fix: Gateway defaults to `min_replicas=2` to prevent ~5 minute 503 outage on pod restart**: The gateway autoscaler defaulted to `min=1`, meaning any pod restart left no replica serving traffic during the 300s readiness delay, causing a guaranteed ~5 minute 503 outage visible to all users. The framework now defaults the gateway to `min=2` so one pod dying never takes it fully down; the surviving pod absorbs traffic immediately while the replacement initialises. Backend services keep `min=1` as their default. The per-app override `[plugins.scale.microservices.services.__gateway__.hpa] min = N` in `jac.toml` is still respected.

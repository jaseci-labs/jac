- **Refactor: `test_mongo_layer123` tests share a single MongoDB and Redis container**: Each test previously spun up and tore down its own `MongoDbContainer` (and in one case a `RedisContainer`), causing the suite to pay container startup cost multiple times. All tests now share lazily-started containers via `_get_mongo()` / `_get_redis()` helpers and reset state between tests with `_reset_mongo()` / `_reset_redis()`, matching the pattern already used in `test_memory_hierarchy`. This significantly reduces total CI wall-clock time for the jac-scale test suite.

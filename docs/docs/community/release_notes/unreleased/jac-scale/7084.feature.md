- **Feature: ship a prebuilt `jac` binary to pods, driven by `[dev]`**: the k8s deploy now resolves a runtime channel from the project's `[dev]` jac.toml stanza and ships a self-contained `jac` binary over the bundle PVC instead of building jaclang. No `[dev]` downloads the latest published release (`stable`); a bare `[dev]` downloads the rolling `dev` prerelease (main); `[dev] jaclang_source` host-builds the local checkout and ships it (`local`); `[dev] repo`/`branch` clones and builds that branch in-pod (`branch`, the only path that still builds). Since scale and the MCP server are built into the binary, stable/dev/local pods do no clone, no plugin install, and no build: they install the shipped binary and run, removing the OOM-prone in-pod jaclang build from the default path.

- **Feat: `MongoBackend.execute_plan` wires FilterSpec pushdown to the QueryPlan IR**: When the runtime detects a `FilterSpec` on a single-hop traversal and `MongoBackend` declares `field_pushdown` + `id_in` capabilities, `refs()` builds a `QueryPlan` and routes it through `execute_plan()` - translating the predicate dict and neighbour `id_in` set into a single `collection.find()` query. Supersedes the removed `query_by_type` duck-type API (use `execute_plan` instead).

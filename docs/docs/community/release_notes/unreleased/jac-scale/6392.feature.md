- **Feature: distributed tracing - in-admin Distributed Traces page (O-02)**. A native span-waterfall view in the jac-scale admin, the generic-span counterpart to the A-05b LLM-trace inspector and the trace sibling of the A-05a Loki logs page. A new `JacAPIServerTraces` mixin (`admin/traces.jac` + impl) exposes `GET /admin/traces/services`, `/admin/traces` (TraceQL search), and `/admin/traces/{trace_id}` (one trace's spans flattened for a waterfall), proxying Tempo's HTTP API with the same env->K8s-DNS->localhost resolver and admin-auth gating as the logs backend; registered on both the monolith server and the microservice gateway. `trace_id` accepts the K-12 dashed UUID and normalizes to the 32-hex form Tempo stores, so a click from the logs page resolves the trace. The React `TracesPage` (under Monitoring) renders a TraceQL search bar (service / errors-only / min-duration / window), a newest-first trace list, and a flame-graph waterfall (one bar per span, positioned by start offset, sized by duration, nested by parent, colored by service). `test_traces_admin.jac` unit-tests the pure helpers: trace-id normalization, TraceQL building, and the Tempo OTLP-JSON -> flat-span-list flatten.

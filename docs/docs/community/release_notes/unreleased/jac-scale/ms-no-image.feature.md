- **Feature: `--no-image` microservice deploy (no Docker build/push)**: `jac start <entry>.jac --scale --no-image` deploys a microservice app to Kubernetes without building or pushing a Docker image. Instead of a built image, the local source tree is tar+gzipped into a ConfigMap, a bootstrap initContainer extracts it and `pip install`s the pinned `jaclang` + `jac-scale` releases into the pod, and each service runs on the generic `python:3.12-slim` base image. This brings microservice mode to parity with the monolith's source-clone path (roadmap N-22) for registry-less clusters. It is a development / preview convenience - two warnings are printed at deploy time recommending a real image for production and `jac setup microservice` for production configuration - and is bounded by the ~1 MiB ConfigMap size limit (oversized source trees fail fast with a clear message). MongoDB/Redis are still provisioned so services share state. Production-grade in-cluster image builds (kaniko is deprecated; buildkit / buildah are the candidates) are tracked as the next roadmap item (N-22b). The default image-build path and all existing deploys are unchanged.

- **Fix: KEDA `TriggerAuthentication` names scoped per-service to prevent credential collision**: `_apply_trigger_auth` and `_build_trigger` derived resource names as `{trigger.name or trigger.type}-trigger-auth` with no service context, so two services sharing a trigger name silently overwrote each other's `TriggerAuthentication` credentials, and two unnamed triggers of the same type in one spec collided the same way. Names are now produced by a shared `_trigger_auth_name` helper as `{service[:20]}-{sha256[:8]}-ta`, scoping each resource to its owning service and differentiating unnamed same-type triggers by their position in the spec.

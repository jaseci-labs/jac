- **Feature: admin Deployments page backed by Prometheus**: The Deployments slot in the admin portal was a placeholder card. Added `JacAPIServerDeployments` mixin exposing `/admin/deployments`, `/admin/deployments/list`, `/admin/deployments/{name}` and `/admin/deployments/{name}/timeseries` which query the in-cluster Prometheus (already provisioned by the monitoring stack alongside kube-state-metrics + node-exporter + cAdvisor) and project the raw PromQL output into a friendly shape: phase counts collapse into `Healthy`/`Starting`/`Broken`/`Unstable` server-side, monitoring-stack pods are hidden from the deployments list, and Prometheus-unreachable falls through to `metrics_available: false` so the UI shows a banner instead of erroring. New `utilities/metrics/prometheus_client.jac` wraps `/api/v1/query` and `/api/v1/query_range` and resolves the Prometheus URL from `K8S_APP_NAME` (honoring `PROMETHEUS_URL` for local dev). `KubernetesTarget` now injects `K8S_APP_NAME` and `K8S_NAMESPACE` on the main jaseci pod so this resolution works without microservice scaffolding. The admin UI ships a new `DeploymentsPage` that renders summary cards, a per-deployment table with traffic-light status badges, and polls every 10s.

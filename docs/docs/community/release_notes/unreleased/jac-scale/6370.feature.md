- **Feature: distributed tracing (OpenTelemetry) for microservice mode (O-02, app side)**. Adds the traces leg of the observability tripod on top of K-12 trace-id propagation and M-14 structured logs. A new `jac_scale.microservices.runtime.tracing` module configures the OpenTelemetry SDK (`TracerProvider` + parent-based ratio sampler + `BatchSpanProcessor` + OTLP/HTTP exporter) once per process and opens one SERVER span per request on both the gateway and each service, with the inbound W3C `traceparent` (forwarded on every proxy hop and `sv_service_call`/`sv_walker_call` RPC) used as the parent so the spans nest into one tree. The K-12 `X-Trace-Id` UUID4 is reused verbatim as the 128-bit OTel trace id, so a request's spans and its log lines (which already carry `trace_id`) share one id and Grafana's trace<->log pivots join correctly. OpenTelemetry is an optional dependency (`jac-scale[tracing]`, guarded through `_optdeps/opentelemetry.jac`); everything degrades to a no-op without it, and the feature is off by default - enable with `[plugins.scale.microservices.tracing] enabled = true` (optional `endpoint` and `sample_ratio` knobs). The id-reconciliation helpers (`uuid_to_trace_id`, `build_traceparent`, `parse_traceparent`) are pure and covered by `test_tracing.jac`. The collection backend (Tempo deploy + Alloy OTLP receiver lane + Grafana datasource) lands separately as O-02.a.

- **Feature: distributed tracing (OpenTelemetry -> Tempo) for microservice mode (O-02)**. Adds the traces leg of the observability tripod on top of K-12 trace-id propagation and M-14 structured logs, with a native in-admin flame graph. **App side:** a new `jac_scale.microservices.runtime.tracing` module configures the OpenTelemetry SDK (`TracerProvider` + parent-based ratio sampler + `BatchSpanProcessor` + OTLP/HTTP exporter) once per process and opens one SERVER span per request on the gateway and each service, with the inbound W3C `traceparent` (forwarded on every proxy hop and `sv_service_call`/`sv_walker_call` RPC) as the parent so spans nest into one tree. The K-12 `X-Trace-Id` UUID4 is reused verbatim as the 128-bit OTel trace id, so a request's spans and its `trace_id`-tagged log lines share one id and Grafana's trace<->log pivots join. OpenTelemetry is an optional dependency (`jac-scale[tracing]`, guarded through `_optdeps/opentelemetry.jac`); everything degrades to a no-op without it, and the feature is off by default - enable with `[plugins.scale.microservices.tracing] enabled = true` (`endpoint`/`sample_ratio` knobs). **Cluster side:** `MonitoringDeployer._deploy_tempo` brings up single-binary Tempo (OTLP receivers + TraceQL API, emptyDir), the Alloy DaemonSet doubles as the OTel collector (an `otelcol.* OTLP->batch->Tempo` lane plus a ClusterIP OTLP Service), Grafana gains a Tempo datasource (nodeGraph + tracesToLogsV2), and the microservice target injects `OTEL_EXPORTER_OTLP_ENDPOINT` into every pod. **Admin UI:** a new `/admin/traces/*` Tempo-proxy endpoint group (`JacAPIServerTraces`, registered on the monolith and the gateway) backs a native Distributed Traces page (`TracesPage.cl.jac`) under Monitoring that renders the span waterfall, the generic-span counterpart to the A-05b LLM-trace inspector. Id-reconciliation helpers are covered by `test_tracing.jac`.

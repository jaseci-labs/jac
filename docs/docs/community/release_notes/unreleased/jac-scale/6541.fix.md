- **Fix: warn when content-addressed image tagging falls back to the mutable tag**. `_retag_with_content_id` degrades to the plain `:dev` tag when `docker inspect`/`docker tag` fails, which silently disables auto-rollout on rebuild and lets a node serve a stale cached image - the exact failure that is hard to diagnose after the fact. It now logs a warning at each fallback point explaining the consequence, instead of degrading silently. No behavior change to the happy path.

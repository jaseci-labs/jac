- **Fix: create the referenced ServiceAccount in microservice deploys**: the microservice target now read-or-creates the configured `service_account_name` ServiceAccount before applying the Deployments, instead of emitting a `serviceAccountName` reference it never creates. Since #6800 added a blocking fleet-readiness wait, a fresh-namespace deploy whose SA was provisioned out-of-band failed hard (the ReplicaSet rejected every pod with "serviceaccount not found", so the fleet never became ready). jac-scale now ensures the SA exists, mirroring the monolith path; `create-if-missing` leaves any caller-managed RBAC/IRSA untouched (#6883).

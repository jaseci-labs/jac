- **Docker Build Pathway (`--build`)**: `jac start app.jac --scale --build` now builds and pushes a Docker image to DockerHub before deploying. If no `Dockerfile` is present, one is auto-generated at `.jac/Dockerfile.gen` (`.jac/` is gitignored so the generated file never appears in your repo). A `.dockerignore` is also created if absent, excluding `jac.local.toml`, `jac.*.toml`, `.env`, and `.jac/`; `.env` is intentionally excluded to prevent secrets from being baked into the image; use `[plugins.scale.secrets]` in `jac.toml` for runtime env vars instead. Sets the correct K8s startup command automatically.
- **Private DockerHub Support**: Automatically creates and manages a K8s image pull secret for private DockerHub images. Secret is updated on re-deploy and deleted on `jac destroy`.
- **Configurable Target Platform**: Added `build_platform` config (default: `linux/amd64`) to control the single target architecture passed to the Docker build. Useful when building on ARM (Apple Silicon) for x86_64 K8s nodes. Configurable via `[plugins.scale.kubernetes]` in `jac.toml`. Note: this sets a single target arch; it is not a multi-arch build.
- **AWS NLB Readiness Polling**: `_wait_for_deployment` polls `read_namespaced_service` until the NLB hostname is assigned, then raises an explicit error if the deadline is exceeded instead of silently skipping the health check. Configurable via `nlb_poll_interval` (default: 10s) and `nlb_timeout` (default: 120s) under `[plugins.scale.kubernetes]` in `jac.toml`.

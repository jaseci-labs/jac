- **Docker Build Pathway (`--build`)**: `jac start app.jac --scale --build` now builds and pushes a Docker image to DockerHub before deploying. Auto-generates a `Dockerfile` (with Bun, `jac install`, and layer-cached `jac.toml`) and `.dockerignore` if not present. The auto-generated `.dockerignore` excludes `jac.local.toml`, `jac.*.toml`, `.env`, and `.jac/` - note that `.env` is intentionally excluded to prevent secrets from being baked into the image; use `[plugins.scale.secrets]` in `jac.toml` for runtime env vars instead. Sets the correct K8s startup command automatically.
- **Private DockerHub Support**: Automatically creates and manages a K8s image pull secret for private DockerHub images. Secret is updated on re-deploy and deleted on `jac destroy`.
- **Cross-Platform Builds**: Added `build_platform` config (default: `linux/amd64`) so ARM machines (Apple Silicon) produce images compatible with x86_64 K8s nodes. Configurable via `[plugins.scale.kubernetes]` in `jac.toml`.
- **AWS NLB Readiness Polling**: `aws_nlb_wait` (default: 10s) is now used as a poll interval rather than a single upfront sleep. `_wait_for_deployment` polls `read_namespaced_service` every `aws_nlb_wait` seconds for up to 120s until the NLB hostname is assigned, then raises an explicit error if the deadline is exceeded instead of silently skipping the health check.

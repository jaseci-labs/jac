- **Fix: in-admin Logs UI on the microservice gateway spun forever on "Loading logs..."**. The microservice gateway lost its `JacAPIServerLogs` inheritance (and the `register_logs_endpoints()` call in `_install_admin_api`) during the M-14/A-05 PR-slicing: A-05c (#6153) correctly dropped the logs wiring as out-of-scope, but A-05a (#6211) only re-added it to the monolith server (`serve.jac`), not the gateway. Result: `/admin/logs?…` requests fell through to the SPA catchall and returned `index.html` (200 HTML), so the React Logs page never got JSON. `MicroserviceGateway` now inherits `JacAPIServerAdmin` + `JacAPIServerLogs` again and `_install_admin_api` registers both endpoint sets before `create_server()` - restoring the EKS-validated state. Regression guard added in `test_gateway.jac`.

- **Fix: Redis L2 cache now invalidated on cascade quarantine**: When `_mongo_quarantine` moved a `NodeAnchor` to quarantine and cascade-quarantined its connected edges, it correctly stripped the orphaned edge IDs from live nodes in MongoDB but did not invalidate those nodes' Redis entries. Any pod that had cached a live node before the quarantine would continue serving a stale entry with dangling edge IDs — even across restarts — causing `EdgeAnchor [<id>] is not a valid reference` on the next walker traversal. Fixed by giving `MongoBackend` a reference to the Redis backend (`_l2_ref` / `set_cache_ref`) and calling `l2.delete()` for the quarantined node, each orphaned edge, and each live node whose `data.edges` was mutated by the cascade.

- **Feature: Zero-config local microservice mode**: `jac start main.jac` now auto-detects microservices and runs them locally with no `jac setup` and no Docker images. When an app splits itself with `sv import from <module>` / `to sv:`, the entry file's directory is scanned for those source markers (comments and docstrings are ignored, so documentation that mentions `sv import` is never miscounted) and each discovered service is given a default `/api/<name>` route, spawned as a subprocess, and fronted by the gateway on `:8000`. A one-line banner reports what was detected and points to `jac setup microservice` for customization. The `enabled` flag under `[plugins.scale.microservices]` is now tri-state: `true` opts in explicitly (routes auto-filled from discovery when none are declared), `false` opts out and stays a monolith, and leaving it unset triggers auto-detection. Apps with no `sv import` (plain monoliths) are unaffected and run exactly as before. The `--scale` remote-deploy path is unchanged and still requires explicit configuration.

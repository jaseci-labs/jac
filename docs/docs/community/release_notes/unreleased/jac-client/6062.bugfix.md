- **Fix: tolerate `undefined` JSX child in `__jacJsx`**: After PR #6042, the per-child guard in `jac-client/jac_client/plugin/impl/client_runtime.impl.jac` only rejected `None` (JS `null`), so any JSX child that evaluated to `undefined` -- a missing prop access, an unspread key, or a slot that early-returned without a value -- crashed at render with `TypeError: Cannot read properties of undefined (reading '__jacUnsafeHtml')`. The guard now skips both `None` and `undefined` before reaching the `unsafe_html` marker check, restoring the pre-#6042 behavior for optional or absent children. A static regression test pins the guard shape so re-narrowing the check is caught at build time rather than at component render.

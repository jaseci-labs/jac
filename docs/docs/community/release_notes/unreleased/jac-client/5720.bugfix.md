- **Fix: `test_desktop_build.jac` no longer fails to compile**: Five tests had docstrings as the first statement inside the `test { ... }` body, which the Jac parser rejects with `error[E0002]: Missing ';'` (the parser also emits `warning[W0060]: Docstrings in Jac go before the declaration`). The compile produced no bytecode and pytest could not import the file. Docstrings moved to the position the W0060 warning indicates - immediately before each `test "..."` declaration, matching the pattern used in `test_cli.jac` and the other `.jac` test suites in the repo. Surfaced by #5694.

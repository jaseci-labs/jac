- **Feat: Configurable PyInstaller Sidecar Build Timeout**: Desktop builds now read `[desktop.bundle] pyinstaller_timeout` (seconds) from `jac.toml`, with a default of `2400` (40 minutes) -- up from the previous hard-coded 10 minutes which was too short for first-run Windows builds, and tighter than an earlier 2-hour bump so genuinely stuck builds fail fast. Override in `jac.toml` when a project's plugin set or host hardware legitimately needs more time. The timeout error message now renders the actual configured value and points users to the jac.toml key. The `jaclang --version` probe used during sidecar preparation also moved from `SUBPROCESS_TIMEOUT_SHORT` (5s) to `SUBPROCESS_TIMEOUT_MEDIUM` (60s), since the first invocation can compile ~20 modules on Windows and exceed 5 seconds.

- **`jac shadcn` top-level command** with six actions: `init`, `add`, `remove`, `list`, `upgrade`, `prune`.
- **Theme flags on `jac create --use shadcn`** so scaffold + theme + install happens in one command. Pre-hook validates flags before any files are written; bad theme aborts cleanly.
- **Declarative theme flags everywhere**: `--style`, `--base-color`, `--theme`, `--font`, `--radius`, `--menu-accent` on `init`, `upgrade`, and `jac create`. 3-level fallback (CLI flag → `[jac-shadcn]` in `jac.toml` → hardcoded default).
- **Client-side validation** with `difflib`-powered "did you mean: …?" suggestions for typos. Tries `GET /options` on the registry first; falls back to a hardcoded list of all valid values.
- **`--all` / `--bare` install-scope flags** on `init` and `jac create --use shadcn`. `--all` installs every component the registry ships; `--bare` installs none. Mutex check in pre-hook.
- **Style switching via `jac shadcn upgrade --style <new>`**: re-fetches every installed component in the new style + persists to `jac.toml`. Resolves the long-standing "style is baked at install time" footgun.
- **`jac-shadcn.lock`**: every install recorded with version + npm deps. Commit to VCS. Drives auto-prune and `upgrade`.
- **Auto-prune on `jac shadcn remove`**: drops any `[dependencies.npm]` entries no remaining installed component still needs. `--keep-deps` opts out.
- **`name@version` pinning** on `jac shadcn add`.
- **`global.css` user-section markers**: user CSS between `/* === jac-shadcn:user BEGIN/END === */` survives every `init` and `upgrade`.
- **`jac shadcn list` / `jac shadcn prune`**: list installed/available; prune orphan npm deps without uninstalling.

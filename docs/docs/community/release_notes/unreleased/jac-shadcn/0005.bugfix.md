- **Fix: `jac create --use shadcn` wrote to an ancestor `jac.toml` when run from a dir nested inside another Jac project**: the `jac create` post-hook was calling `get_config()` which returned a stale process-lifetime cache pointing at the ancestor project (populated earlier in the same `jac create` invocation when the new project's `jac.toml` didn't exist yet). Fixed by passing `start_path=project_dir, force_discover=True` to bypass the cache. Includes a regression test in `test_init_shadcn_theme.jac` that builds a parent + child layout and asserts only the child is written.
- **Fix: scaffolded `main.jac` failed to run with `jac start`**: the template referenced a non-existent sibling module (`import from .app { app }`) which raised `attempted relative import with no known parent package`. Replaced with a self-contained welcome page (no component imports) that works with any `--all` / `--bare` / default install set.

# LEARNING-RECORD-FORMAT.md

Learning records capture non-obvious lessons and key insights — loosely equivalent
to architectural decision records. They drive the zone of proximal development.

Titled `0001-<dash-case-name>.md`, incrementing each time.

## Format

```markdown
# 0001 — <short title>

**Date:** <YYYY-MM-DD>
**Type:** <insight | preference | mission-change | misconception-corrected>

## What was learned

<2-5 sentences: the non-obvious thing>

## Why it matters

<1-3 sentences: how this shapes future lessons or the mission>

## Evidence

<what grounded this — a resource, a code path, a test result, a user statement>
```

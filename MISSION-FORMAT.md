# MISSION-FORMAT.md

`MISSION.md` captures the *reason* the user is interested in the topic. It grounds
all teaching. Update it when the mission evolves (and record the change in a
learning record).

## Format

```markdown
# Mission

<one or two sentences: what the user wants and why>

## Context

<2-4 bullets: relevant background — current skill level, related projects,
constraints, what they already know>

## Success looks like

<2-3 bullets: concrete signals that the user has achieved their goal>

## Out of scope

<1-2 bullets: explicitly what this mission is NOT about, to keep lessons focused>
```
